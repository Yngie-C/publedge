# PDF 내보내기와 파이프라인 완성: 네 번째 스킬

드디어 마지막 스킬이에요. 이번 챕터를 마치면 자소서 파이프라인이 완전히 완성돼요. JD를 넣으면 완성된 PDF가 나오는, 진짜 나만의 자동화 도구가 생기는 거예요.

짧게 돌아보면 이렇게 왔어요. 1장에서 `hello.md`로 스킬의 기본을 배웠고, 3장에서 JD를 분석해 JSON으로 저장했어요. 4장에서는 그 JSON을 바탕으로 맞춤형 자소서를 초안으로 만들었죠. 5장에서는 AI가 만든 내용 중 사실과 다른 부분을 자동으로 검토했어요. 그리고 이번 6장에서 최종 결과물인 PDF를 내보내면, 파이프라인의 모든 단계가 완성돼요.

이번 챕터에서 새로 배우는 것은 딱 하나예요. **Claude Code가 터미널 명령어를 직접 실행할 수 있다**는 것. 이 개념만 이해하면, 스킬 파일을 만드는 것 자체는 지금까지와 완전히 똑같아요.

---

## 새로운 개념: Claude Code가 명령어를 실행한다고요?

지금까지 Claude Code는 주로 파일을 읽고 쓰는 작업을 해왔어요. 1장에서 `hello.md` 스킬을 만들었고, 이후 JD를 분석하고, 자소서를 생성하고, 환각을 검토했죠. 이 모든 과정은 "파일을 읽어서 다른 파일을 써요"라는 패턴이었어요.

이번에는 한 단계 더 나아가요. Claude Code는 **터미널 명령어를 직접 실행**할 수도 있어요.

"이 마크다운 파일을 PDF로 변환해줘"라고 하면, Claude는 변환 도구를 실행하는 것까지 직접 처리해요. 코드를 짤 필요도 없고, 터미널 명령어를 외울 필요도 없어요. 스킬 파일 안에 "이 명령어를 실행하세요"라고 적어두기만 하면, Claude가 그 명령어를 실행해요.

이게 왜 강력하냐면, 세상에 있는 수많은 CLI 도구들을 Claude가 대신 사용할 수 있게 되기 때문이에요. PDF 변환뿐 아니라, 이미지 압축, 파일 형식 변환, 코드 포매팅 같은 작업도 스킬 하나로 자동화할 수 있어요.

조금 더 구체적으로 설명하면, 지금까지 만든 스킬들은 Claude가 "생각해서 텍스트를 써주는" 역할을 했어요. JD를 읽고 분석 결과를 JSON으로 쓰거나, 프로필을 읽고 자소서를 써주는 식이었죠. 이번 스킬은 달라요. Claude가 "실제 프로그램을 실행해서 파일을 변환하는" 역할을 해요. 이 차이가 Claude Code를 단순한 글쓰기 도구가 아니라, 진짜 자동화 도구로 만드는 핵심이에요.

> **NOTE:** Claude Code가 터미널 명령어를 실행하기 전에는 항상 "이 명령어를 실행해도 될까요?"라고 물어봐요. 여러분이 명시적으로 허용해야만 실행하니까 안심하세요. Claude가 멋대로 뭔가를 실행하는 일은 없어요.

이 승인 절차가 처음에는 번거롭게 느껴질 수 있지만, 실제로는 굉장히 중요한 안전장치예요. 어떤 명령어가 실행되는지 항상 확인할 수 있거든요. Claude가 실행하려는 명령어가 예상과 다르다면 그 자리에서 거부할 수 있어요.

## md-to-pdf 설치하기

PDF 변환에는 `md-to-pdf`라는 npm 패키지를 사용해요. 이 도구는 마크다운 파일을 깔끔한 PDF로 바꿔줘요. 스타일도 제법 예쁘고, 한글도 잘 지원해요. pandoc 같은 다른 도구들도 있지만, `md-to-pdf`는 Node.js만 있으면 바로 설치할 수 있어서 이 책에서 쓰기에 딱 맞아요.

### 설치 명령어

터미널을 열고 아래 명령어를 입력하세요.

```bash
npm install -g md-to-pdf
```

설치가 완료됐는지 확인하려면 이렇게 해요.

```bash
md-to-pdf --version
```

버전 번호가 출력되면 설치 성공이에요. 예를 들어 `5.4.0` 같은 숫자가 나오면 돼요.

> **NOTE:** 처음 `md-to-pdf`를 실행하면 내부적으로 Chromium(크롬 브라우저 엔진)을 자동으로 다운로드해요. 약 300MB 정도 되니까, Wi-Fi 환경에서 설치하는 걸 권장해요. 이 다운로드는 최초 1회만 발생하고, 이후에는 바로 실행돼요. 다운로드 중에 터미널이 멈춘 것처럼 보일 수 있는데, 그냥 기다리면 돼요.

> **TIP:** `npm install -g` 명령어가 낯설지 않죠? Claude Code를 처음 설치할 때(0장) 사용했던 방법과 똑같아요. `-g`는 global의 약자로 "내 컴퓨터 전체에서 쓸 수 있게 설치해줘"라는 의미예요. `-g` 없이 설치하면 현재 폴더에서만 쓸 수 있어서, 반드시 `-g`를 붙여야 해요.

### md-to-pdf가 설치 안 된다면?

간혹 npm 권한 문제나 네트워크 문제로 설치가 안 되는 경우가 있어요. `EACCES: permission denied` 같은 오류가 뜬다면 권한 문제예요. 이럴 때는 명령어 앞에 `sudo`를 붙여보세요.

```bash
sudo npm install -g md-to-pdf
```

그래도 안 된다면 브라우저를 이용한 대안이 있어요.

1. VS Code에서 자소서 마크다운 파일을 열어요
2. 오른쪽 상단의 미리보기 버튼을 클릭해서 렌더링된 화면을 열어요
3. 미리보기 화면에서 마우스 오른쪽 버튼 클릭 → "인쇄"를 선택해요
4. 프린터 대신 "PDF로 저장"을 선택해요

완벽하진 않지만, 급할 때는 이 방법으로도 제출 가능한 PDF를 만들 수 있어요.

## /export-pdf 스킬 만들기 — build-then-reveal

이번 스킬도 이전 챕터들과 같은 방식으로 만들어요. 먼저 스킬이 무엇을 해야 하는지 이해하고, 그다음에 전체 파일을 작성해요.

### 핵심 먼저 이해하기

`/export-pdf` 스킬은 세 가지 일을 해요.

**첫째, 변환할 파일을 받아요.**
`$ARGUMENTS`에 마크다운 파일 경로가 들어와요. 예를 들어 `/export-pdf output/resume-techcorp-backend.md`라고 입력하면, `output/resume-techcorp-backend.md`가 `$ARGUMENTS`가 돼요. 이전 챕터들에서 써온 방식과 완전히 똑같아요.

**둘째, md-to-pdf로 PDF를 만들어요.**
Claude에게 "이 CLI 도구를 실행해줘"라고 지시하는 거예요. 스킬 파일 안에 실행할 명령어를 적어두면, Claude가 승인 요청 후 실행해요. 출력 폴더는 `output/pdf/`로 고정해요. 같은 경로에 항상 저장되니까 나중에 찾기도 쉬워요.

**셋째, 에러를 처리해요.**
`md-to-pdf`가 설치되지 않은 경우, 설치 방법을 안내해줘요. 스킬이 "무조건 실행"만 하는 게 아니라 상황에 맞게 반응하는 거예요. 이처럼 스킬 파일에 조건을 적어두면 Claude가 그 조건에 맞게 행동해요.

### 전체 스킬 파일 작성

`.claude/commands/` 폴더에 `export-pdf.md` 파일을 만들어요.

~~~markdown
# .claude/commands/export-pdf.md

$ARGUMENTS에 자소서 markdown 파일 경로가 주어집니다.

작업:
1. output/pdf/ 폴더가 없으면 생성하세요.
2. md-to-pdf 명령어로 PDF로 변환하세요:
   ```bash
   md-to-pdf {파일경로} --dest output/pdf/
   ```
3. md-to-pdf가 설치되어 있지 않으면 사용자에게 아래 명령어로 설치하도록 안내하세요:
   ```bash
   npm install -g md-to-pdf
   ```
4. 변환이 완료되면 생성된 PDF 파일의 전체 경로를 알려주세요.

대안: md-to-pdf 설치가 어렵다면 브라우저에서 파일을 열고 Ctrl+P(macOS: Cmd+P) → "PDF로 저장"으로도 변환할 수 있어요.
~~~

짧고 단순하죠? 지금까지 만든 스킬 중에서 가장 짧아요. 그런데 이 짧은 파일 안에 중요한 설계가 담겨 있어요.

명령어를 직접 실행하는 게 아니라, Claude에게 "이 명령어를 실행하세요"라고 위임하는 거예요. 덕분에 Claude는 실행 전에 파일 경로가 올바른지 확인하고, 폴더가 없으면 먼저 만들어요. 그리고 실행 결과가 이상하면 사용자에게 설명해줘요. 단순한 명령어 실행이 아니라, Claude가 판단하면서 실행하는 거예요.

스킬 파일을 저장했으면 이제 실행해볼게요.

## 스킬 실행 & PDF 확인

Claude Code 채팅창에 아래처럼 입력해요.

```bash
/export-pdf output/resume-techcorp-backend.md
```

Claude가 이렇게 응답할 거예요.

> "output/pdf/ 폴더를 생성하고, 아래 명령어를 실행할게요. 허용하시겠습니까?"
>
> `md-to-pdf output/resume-techcorp-backend.md --dest output/pdf/`

**"허용"을 클릭**하면 변환이 시작돼요. 터미널에서 `md-to-pdf`가 실행되고, 진행 상황이 표시돼요.

잠시 후 Claude가 결과를 알려줘요.

> "PDF 변환이 완료됐어요. 파일 위치: `output/pdf/resume-techcorp-backend.pdf`"

`output/pdf/` 폴더를 열어보면 PDF 파일이 생겨 있어요. 파일을 더블클릭해서 열어봐요.

자소서가 깔끔하게 포맷된 PDF로 변환된 걸 볼 수 있어요. 헤딩은 굵게 표시되고, 글머리 기호와 본문 텍스트가 모두 제대로 렌더링됐어요. 한글도 깨지지 않아요.

처음 실행할 때 Chromium을 다운로드하는 경우 1~2분 정도 걸릴 수 있어요. 이후 실행에서는 10초 내외로 빠르게 변환돼요.

PDF가 열렸을 때 레이아웃이 어색하다면, 자소서 마크다운 파일의 구조를 점검해보세요. 헤딩(`##`, `###`)을 잘 활용했는지, 긴 단락을 적절히 나눴는지 확인하면 대부분 깔끔하게 나와요. 마크다운이 잘 정리돼 있으면 PDF도 자동으로 예쁘게 나오는 게 `md-to-pdf`의 장점이에요.

> **TIP:** PDF 스타일이 마음에 안 들면, `md-to-pdf`의 `--stylesheet` 옵션으로 CSS 파일을 지정할 수 있어요. 예를 들어 폰트를 바꾸거나, 여백을 조절하거나, 줄 간격을 늘리는 것도 가능해요. 하지만 지금 단계에서는 기본 스타일로도 충분히 제출 가능한 수준이니, 일단 기본 설정으로 써보세요.

## 전체 파이프라인 실행해보기!

네 개의 스킬이 모두 완성됐어요. 이제 처음부터 끝까지 전체 흐름을 한 번 실행해볼게요. 새 회사에 지원한다고 가정하고, `newcompany-frontend`라는 프론트엔드 포지션에 지원하는 시나리오로 진행해요.

### Step 1 — JD 파일 준비

채용공고 페이지에서 JD 내용을 복사해서 새 파일을 만들어요. Claude Code에서 아래처럼 부탁하면 돼요.

> "data/jd/newcompany-frontend.md 파일을 만들고, 아래 내용을 저장해줘."

그다음에 복사한 JD 텍스트를 붙여넣으면, Claude가 파일을 만들어줘요. 직접 파일을 만들어도 돼요. 어느 쪽이든 상관없어요.

> **TIP:** JD 파일 이름은 `회사명-직군.md` 형식으로 통일하면 나중에 관리하기 편해요. `kakao-frontend.md`, `naver-backend.md`처럼요. 파일 이름만 봐도 어느 회사 어느 포지션인지 바로 알 수 있어요.

### Step 2 — JD 분석

```bash
/collect-jd data/jd/newcompany-frontend.md
```

Claude가 JD를 읽고, 이 회사가 원하는 핵심 역량, 기술 스택, 우대 사항을 분석해서 구조화된 JSON 파일로 저장해요. 이 JSON이 이후 모든 단계의 기반이 돼요.

결과물: `data/analysis/newcompany-frontend.json`

분석에는 보통 1분 이내가 걸려요.

### Step 3 — 자소서 생성

```bash
/generate-resume data/analysis/newcompany-frontend.json
```

Claude가 분석 결과와 내 프로필(`data/my-profile.md`)을 결합해서, 이 회사에 맞춘 자소서 초안을 만들어요. JD에서 강조한 키워드가 자소서에 자연스럽게 녹아들어요.

결과물: `output/resume-newcompany-frontend.md`

### Step 4 — 환각 검토 및 품질 개선

```bash
/review-resume output/resume-newcompany-frontend.md data/analysis/newcompany-frontend.json
```

Claude가 자소서를 검토하고, 사실과 다른 내용이나 과장된 표현을 지적해줘요. 검토 결과를 보고 필요한 부분을 수정해요.

직접 파일을 열어서 고쳐도 되고, Claude에게 "3번 항목의 기간을 2021-2023으로 수정해줘"처럼 부탁해도 돼요. 수정이 끝나면 `/review-resume`을 한 번 더 돌려서 다시 확인해도 좋아요.

결과물: `output/review-resume-newcompany-frontend.md`

### Step 5 — PDF 내보내기

```bash
/export-pdf output/resume-newcompany-frontend.md
```

Claude가 md-to-pdf를 실행해서 PDF를 생성해요. 실행 승인 요청이 뜨면 허용을 클릭하면 돼요.

결과물: `output/pdf/resume-newcompany-frontend.pdf`

이게 전부예요. JD를 붙여넣고 다섯 번의 명령어를 실행하면, 제출 가능한 PDF가 나와요.

5개 명령어를 한눈에 보면 이렇게 돼요.

```bash
# 1. JD 파일 준비 (한 번만)
# data/jd/newcompany-frontend.md 생성

# 2. JD 분석
/collect-jd data/jd/newcompany-frontend.md

# 3. 자소서 초안 생성
/generate-resume data/analysis/newcompany-frontend.json

# 4. 환각 검토 (수정 후 반복 가능)
/review-resume output/resume-newcompany-frontend.md data/analysis/newcompany-frontend.json

# 5. PDF 내보내기
/export-pdf output/resume-newcompany-frontend.md
```

> **TIP:** 새 회사에 지원할 때마다 **Step 1의 JD 파일만 바꾸면** 나머지 흐름은 똑같아요. 10개 회사에 지원해도 10번 이 흐름을 반복하면 되고, 매번 그 회사에 최적화된 자소서가 나와요. 한 번 파이프라인을 세팅해두면 이후에는 반복 작업이 거의 없어요.

## 최종 프로젝트 구조 확인

챕터를 마친 지금, 여러분의 프로젝트 폴더는 이런 모습이에요.

```
my-resume-pipeline/
├── CLAUDE.md
├── resume-v1.md
├── data/
│   ├── jd/
│   │   └── techcorp-backend.md
│   ├── analysis/
│   │   └── techcorp-backend.json
│   └── my-profile.md
├── output/
│   ├── resume-techcorp-backend.md
│   ├── review-resume-techcorp-backend.md
│   └── pdf/
│       └── resume-techcorp-backend.pdf    ← 최종 결과물!
└── .claude/
    └── commands/
        ├── hello.md
        ├── collect-jd.md
        ├── generate-resume.md
        ├── review-resume.md
        └── export-pdf.md                  ← 이번 챕터에서 추가!
```

0장에서 빈 폴더로 시작했던 프로젝트가, 이제는 완성된 자동화 파이프라인이 됐어요. `.claude/commands/` 안에 스킬 파일이 다섯 개 생겼어요. 처음에 `hello.md` 하나로 시작했던 게 엊그제 같은데, 이제는 취업 준비 전체를 아우르는 도구가 완성됐어요. 앞으로 새 회사에 지원할 때마다 이 폴더를 열면, 필요한 모든 도구가 이미 준비되어 있을 거예요.

## 축하해요! 파이프라인이 완성됐어요! 🎉

정말로 해냈어요. 쉽지 않았을 텐데, 여기까지 따라오신 거 진심으로 축하드려요.

이 책을 시작할 때 "AI 도구를 사용하는 것"과 "나만의 AI 파이프라인을 만드는 것"은 완전히 다른 이야기라고 했죠. 지금 여러분은 후자를 해낸 거예요. ChatGPT에 자소서를 붙여넣고 "이거 고쳐줘"라고 하는 게 아니라, 처음부터 끝까지 자동으로 돌아가는 나만의 시스템을 직접 만든 거예요.

생각해보면 꽤 많은 것들을 배웠어요. 터미널을 처음 열었을 때의 낯선 느낌, 스킬 파일이 제대로 인식이 안 됐을 때의 당황스러움, Claude가 예상과 다른 결과를 냈을 때의 답답함. 그걸 하나씩 극복하면서 여기까지 왔어요. 그 과정 자체가 이미 값진 경험이에요.

지금 여러분이 할 수 있는 것들을 정리해볼게요.

**첫째**, Claude Code를 설치하고 자유롭게 사용할 수 있어요. 프롬프트를 입력하고, 파일을 주고받고, 결과를 확인하는 흐름이 이제 자연스럽게 느껴지죠? 처음에 터미널을 열었을 때의 낯섦이 이제는 많이 사라졌을 거예요.

**둘째**, 나만의 AI 스킬(커스텀 명령어)을 만들 수 있어요. `.claude/commands/` 폴더에 마크다운 파일 하나를 추가하는 것만으로, 반복 작업을 자동화할 수 있어요. 자소서뿐 아니라 어떤 반복 작업이든 스킬로 만들 수 있어요. 이 원리를 이해했으면, 앞으로 어떤 도구든 직접 만들 수 있어요.

**셋째**, JD 맞춤형 자소서를 자동으로 생성할 수 있어요. 더 이상 채용공고를 읽고 머리를 싸매며 "이 회사는 어떤 사람을 원하는 걸까?" 고민하지 않아도 돼요. Claude가 분석해줘요. 여러분은 그 결과를 검토하고 필요한 부분만 손보면 돼요.

**넷째**, 환각을 검출하고 품질을 높일 수 있어요. AI가 만든 콘텐츠를 그냥 믿는 게 아니라, 검증 단계를 파이프라인에 포함시키는 방법을 알게 됐어요. 이건 자소서를 넘어서, AI를 쓰는 모든 상황에 적용되는 핵심 원칙이에요. AI 도구를 "잘 쓰는 것"과 "안전하게 쓰는 것"은 다르고, 여러분은 둘 다 배웠어요.

**다섯째**, 완성된 자소서를 PDF로 내보낼 수 있어요. Claude Code가 터미널 명령어까지 실행할 수 있다는 것도 배웠어요. 이제 어떤 CLI 도구든 Claude에게 실행을 맡길 수 있어요. 이 원리를 확장하면, 세상에 있는 수많은 도구들을 Claude 스킬로 엮어서 더 큰 파이프라인을 만들 수 있어요.

### 더 나아가고 싶다면

파이프라인은 완성됐지만, 여기서 멈출 필요는 없어요. 뒤에 준비된 부록에서 더 많은 것들을 배울 수 있어요.

**부록 A — 문제가 생겼을 때 해결법**
Claude Code가 이상하게 동작하거나, 스킬이 제대로 실행되지 않을 때 확인할 체크리스트예요. "스킬이 인식이 안 돼요", "Claude가 엉뚱한 파일을 수정했어요", "터미널 명령어가 실패했어요" 같은 상황별 해결법을 정리했어요. 설치 문제부터 프롬프트 디버깅까지, 흔히 겪는 문제들을 다뤄요.

**부록 B — 다양한 프롬프트 템플릿**
이 책에서 만든 4개의 스킬 외에, 다른 상황에 응용할 수 있는 프롬프트 템플릿 모음이에요. 포트폴리오 정리, 면접 준비 Q&A 생성, 자기소개서 변형, 영문 이력서 번역 등 취업 준비 전반에 활용할 수 있는 스킬 아이디어를 담았어요. 복사해서 바로 쓸 수 있어요.

**부록 C — hooks, MCP, cron 등 고급 기능**
파이프라인을 더 정교하게 만들고 싶은 분들을 위한 내용이에요. 특정 파일이 저장될 때 자동으로 스킬이 실행되게 하거나, 외부 API와 연동하거나, 스케줄에 따라 자동 실행되게 하는 방법을 소개해요. 개발 경험이 없어도 따라올 수 있도록 최대한 쉽게 썼어요. 이 기능들을 활용하면 파이프라인이 "실행할 때만 돌아가는 도구"에서 "항상 작동하는 자동화 시스템"으로 진화해요.

책을 끝까지 읽어주셔서 감사해요. 이 파이프라인을 만드는 동안 배운 것들은 자소서 너머로도 쓸 수 있어요. 반복적인 일을 자동화하는 원리, AI에게 명확하게 지시하는 방법, 결과물을 검증하는 습관. 이 모든 게 앞으로 AI 도구를 쓸 때마다 빛을 발할 거예요.

한 가지만 당부드리고 싶어요. 파이프라인을 만들었다고 해서 자소서를 AI에게만 맡기지는 마세요. 이 파이프라인은 여러분의 경험과 생각을 더 빠르고 정확하게 표현하도록 돕는 도구예요. 최종 판단과 수정은 항상 여러분의 몫이에요. AI가 만든 초안을 꼼꼼히 읽고, 내 목소리가 담겼는지 확인하고, 필요하면 직접 고치세요. 그렇게 AI와 사람이 함께 만든 자소서가, 결국 가장 좋은 자소서예요.

여러분이 만든 파이프라인이 취업 준비에 실질적인 도움이 되길 바랄게요. 좋은 결과 있으시길 진심으로 응원해요. 🎉
