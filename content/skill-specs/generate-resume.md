# /generate-resume Skill Specification

## 용도
JD 분석 결과와 사용자의 경력기술서를 바탕으로 맞춤형 자소서를 생성하는 Claude Code 커스텀 커맨드.

## 파일 위치
`.claude/commands/generate-resume.md`

## Skill 파일 내용

아래가 `.claude/commands/generate-resume.md`에 들어갈 실제 프롬프트 내용이에요:

---

당신은 한국 취업 시장에 정통한 취업 컨설턴트입니다.

### 입력
$ARGUMENTS 에 JD 분석 결과 JSON 파일 경로가 주어집니다.
해당 파일과 `data/my-profile.md`를 함께 읽어주세요.

### 자소서 작성 원칙
1. JD의 **핵심 키워드**를 자연스럽게 포함할 것
2. **구체적 수치와 경험**을 사용할 것 (예: "3개월간", "팀원 4명과")
3. 각 항목 **300-500자** 분량
4. **지원동기**에는 회사 가치관과 나의 가치관 연결
5. **직무역량**에는 기술 스택 매칭
6. 문장은 **간결하고 능동태**로

### 자소서 구조
다음 4개 항목으로 자소서를 작성하세요:

#### 1. 지원동기
- 왜 이 회사인지, 왜 이 직무인지
- JD의 `culture`와 나의 가치관 연결

#### 2. 직무역량
- JD의 `required`와 `tech_stack`에 매칭되는 내 경험
- 프로젝트/과목/활동 구체적으로

#### 3. 성장과정
- 문제를 해결한 경험 (STAR 기법: 상황-과제-행동-결과)
- 팀워크 또는 자기주도 성장 에피소드

#### 4. 입사 후 포부
- 단기(1년) / 중기(3년) 목표
- JD의 `preferred` 역량 학습 계획

### 출력
완성된 자소서를 `output/` 폴더에 저장하세요.
파일명: `resume-{회사명영문}-{직무명영문}.md`

저장 후 각 항목의 글자 수를 요약해주세요.

---

## 사용 예시

```
/generate-resume data/analysis/techcorp-backend.json
```

## 입력 요구사항
- `/collect-jd`로 생성된 분석 결과 JSON 파일
- `data/my-profile.md` — 사용자의 경력기술서 (필수!)

## 출력물
- `output/resume-{회사}-{직무}.md` — 맞춤형 자소서
- 터미널에 항목별 글자 수 요약

## 설계 의도
- **경력기술서 참조**: `data/my-profile.md`를 source of truth로 사용하여 환각(hallucination) 방지
- **STAR 기법**: 구조화된 경험 서술로 면접 대비까지 연결
- **글자 수 제한**: 한국 자소서의 일반적 항목별 분량(300-500자)을 가이드라인으로 제공
- **영문 파일명**: Windows 터미널 인코딩 이슈 방지
