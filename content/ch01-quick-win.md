# 첫 자소서 초안, 5분 만에 만들기

## 이번 챕터의 목표

이번 챕터는 단 하나의 목표를 가지고 있어요.

**지금 당장, 손으로 직접, 자소서 초안을 만들어보는 것.**

이론 설명은 없어요. 설치 이야기도 없어요. 그냥 바로 해봐요. Claude Code를 열고, 5분 뒤엔 여러분의 이름이 들어간 자소서 초안이 파일로 저장되어 있을 거예요.

코딩 지식? 필요 없어요. 그냥 대화하면 돼요.

---

## Claude Code 시작하기

터미널을 열어요. macOS라면 `Cmd + Space`를 눌러 Spotlight에서 "터미널"을 검색하거나, Windows라면 시작 메뉴에서 Windows Terminal을 검색해서 열어주세요.

열렸으면 아래 명령어를 입력해요.

```bash
mkdir my-resume-pipeline
cd my-resume-pipeline
claude
```

`mkdir`은 폴더를 만드는 명령어예요. `cd`는 그 폴더 안으로 들어가는 명령어고요. 마지막으로 `claude`를 치면 Claude Code가 시작돼요.

잠깐 기다리면 이런 화면이 나타나요.

```
╭─────────────────────────────────────╮
│ Claude Code                         │
│ /Users/yourname/my-resume-pipeline  │
╰─────────────────────────────────────╯

>
```

> **NOTE:** Claude Code가 설치되어 있어야 해요. Claude Pro 또는 Max 구독이 필요해요. 설치가 처음이라면 0장을 먼저 확인하세요.

## 자소서 초안 요청하기

`>` 뒤에 프롬프트를 입력하면 돼요. 아래 예시를 그대로 복사해도 되지만, **괄호 안의 내용은 본인 정보로 꼭 바꿔주세요!**

```
나는 컴퓨터공학 졸업 예정이고, 테크코프(가상 기업)의 백엔드 개발자에 지원하려고 해.
자기소개서 초안을 만들어줘.
항목은 지원동기, 직무역량, 성장과정, 입사 후 포부 4개로 나눠줘.
각 항목은 500자 내외로 써줘.
```

Enter를 누르면 Claude가 생각하기 시작해요. 잠깐 기다리면 자소서 초안이 쭉 출력돼요.

처음 보면 이런 느낌이에요.

```
## 지원동기

저는 대학교 4년간 컴퓨터공학을 전공하며 백엔드 시스템에 깊은 관심을 가져왔습니다.
특히 대용량 트래픽을 처리하는 분산 시스템 설계에 매력을 느꼈고, 테크코프가
국내 최대 규모의 실시간 데이터 처리 인프라를 운영한다는 점에서 입사를 희망하게
되었습니다...

## 직무역량

...
```

> **NOTE:** AI 응답은 매번 달라요. 책에 나온 예시와 100% 같지 않아도 완전히 정상이에요. 중요한 건 구조와 흐름이지, 똑같은 문장이 아니에요.

어때요? 30초도 안 됐는데 그럴싸한 자소서 초안이 나왔죠?

---

## 초안 다듬기: 후속 프롬프트

사실 첫 번째 결과가 완벽할 거라고 기대하지 않아도 돼요. Claude와의 대화는 **이어서 계속**할 수 있어요. 첫 초안을 받은 다음, 바로 이렇게 이어가봐요.

**더 구체적으로 만들고 싶을 때:**

```
좀 더 구체적인 프로젝트 경험을 넣어서 직무역량 부분을 다시 써줘.
예를 들어 Spring Boot로 REST API를 만든 경험이나, MySQL 쿼리 최적화 경험을 포함해줘.
```

**특정 항목을 강화하고 싶을 때:**

```
지원동기 부분을 더 강화해줘.
테크코프가 최근 발표한 AI 추천 시스템 프로젝트에 관심이 생겼다는 내용을 넣어줘.
```

**글자 수를 맞춰야 할 때:**

```
각 항목을 정확히 300자 이내로 줄여줘.
핵심 내용은 유지하고, 불필요한 수식어를 제거해줘.
```

> **TIP:** 프롬프트를 구체적으로 쓸수록 결과가 확실히 좋아져요. "잘 써줘"라고 하면 Claude도 뭘 원하는지 몰라요. 대신 "지원동기에 팀 프로젝트에서 리더 역할을 맡은 경험을 넣어줘"처럼 구체적으로 말해봐요. 같은 요청인데 결과물의 질이 완전히 달라져요.

두세 번 주고받다 보면 점점 여러분 스타일에 맞는 문장이 나오기 시작해요. 이게 Claude와 대화하는 방식이에요.

---

## 파일로 저장하기

마음에 드는 버전이 나왔으면, 이제 파일로 저장해봐요. Claude에게 이렇게 부탁하면 돼요.

```
이 자소서 내용을 resume-v1.md 파일로 저장해줘.
```

Claude가 파일을 직접 만들어줘요. 이렇게요.

```
파일 resume-v1.md를 생성했어요.

my-resume-pipeline/
  └── resume-v1.md
```

저장이 완료됐는지 확인해볼까요? Claude 밖에서, 즉 새 터미널 탭을 열어서 아래 명령어를 입력해봐요.

**macOS:**
```bash
cat ~/my-resume-pipeline/resume-v1.md
```

**Windows:**
```bash
type C:\Users\yourname\my-resume-pipeline\resume-v1.md
```

아까 Claude가 써준 자소서 내용이 그대로 출력되면 성공이에요!

> **TIP:** `Cmd + T` (macOS) 또는 `Ctrl + Shift + T` (Windows Terminal)로 터미널에 새 탭을 열 수 있어요. Claude Code를 종료하지 않고도 파일을 확인할 수 있어요.

## 현재 프로젝트 상태

지금까지 한 것들을 정리해보면 이래요.

```
my-resume-pipeline/
  resume-v1.md          ← 첫 자소서 초안!
```

딱 하나의 파일이지만, 이게 시작이에요.

## 잠깐, 이게 끝? — 한계 인식

방금 만든 자소서, 생각보다 괜찮아 보이죠? 맞아요. Claude는 문장을 꽤 잘 써요. 근데 솔직하게 말해볼게요.

**이 방식엔 문제가 있어요.**

**문제 1: 매번 처음부터 설명해야 해요.**

지금은 "나는 컴퓨터공학 전공이고..."를 직접 입력했어요. 다음에 또 자소서를 만들 때도, 그 다음에도 똑같이 입력해야 해요. 50곳에 지원한다면? 50번 타이핑?

**문제 2: JD(채용공고)를 분석하지 않았어요.**

회사마다 원하는 인재상이 달라요. "협업 능력"을 강조하는 곳도 있고, "주도적 문제 해결"을 원하는 곳도 있어요. 지금 만든 자소서엔 그런 맞춤 키워드가 없어요. 그냥 일반적인 자소서예요.

**문제 3: 검증이 없어요.**

Claude는 그럴싸하게 쓰는 게 특기예요. 근데 가끔 여러분이 한 적 없는 경험을 넣기도 해요. 지금 방식으론 그걸 잡아내기가 쉽지 않아요.

**문제 4: 100곳에 지원한다면?**

이 프로세스를 100번 반복하는 건 상상만 해도 피곤해요. 자동화 없이는 규모를 늘리기가 힘들어요.

---

만약 이 네 가지 문제를 전부 해결할 수 있다면 어떨까요?

- 내 정보를 한 번만 저장해두면, 매번 새 자소서를 만들 때 자동으로 불러와요
- JD를 붙여 넣으면 Claude가 분석해서 맞춤 키워드를 넣어줘요
- 내가 실제로 한 경험만 쓰도록 자동으로 체크해줘요
- 지원할 회사가 늘어나도 몇 초 만에 새 버전을 뽑아낼 수 있어요

이게 바로 이 책에서 만들 **자소서 파이프라인**이에요.

다음 챕터부터 이 문제들을 하나씩 해결해 나가요. 제일 먼저 할 건 "내 정보 저장소"를 만드는 거예요. 나에 대한 모든 정보를 한 곳에 정리해두면, Claude가 언제든 꺼내서 쓸 수 있어요.

준비됐어요? 계속 가봐요!
