# 클로드 코드, 이것만 알면 된다

1장에서 Claude Code를 처음 써봤죠? 명령어 한 줄로 자소서 초안이 파일로 뚝 생겨났어요. 신기하기도 하고, "이게 어떻게 된 거지?" 싶기도 했을 거예요.

이번 장에서는 그 원리를 이해해요. 딱 3가지만 알면 돼요. 이 3가지를 이해하면 3장부터 만들 4개의 스킬을 막힘 없이 따라할 수 있어요.

---

## Claude Code가 뭔가요?

Claude Code는 **CLI(명령줄) 기반 AI 어시스턴트**예요. CLI는 Command Line Interface의 줄임말로, 쉽게 말하면 "텍스트로 명령어를 입력하는 프로그램"이에요. 터미널에서 `claude`라고 입력하면 시작되는 그거요.

### ChatGPT나 웹 Claude와 뭐가 달라요?

ChatGPT나 claude.ai(웹 Claude)도 AI인데, 뭐가 다를까요?

가장 직관적인 비유를 하나 드릴게요.

> ChatGPT나 웹 Claude가 **카카오톡으로 연결된 천재 친구**라면,
> Claude Code는 **그 친구가 내 컴퓨터 앞에 직접 앉아있는 거**예요.

카카오톡으로 "이 문서 고쳐줘"라고 하면 친구가 고쳐준 내용을 메시지로 보내줘요. 내가 직접 받아서 파일에 붙여넣어야 하죠. 번거롭고, 긴 내용이면 여러 번에 나눠서 받아야 할 때도 있어요.

그런데 그 친구가 내 컴퓨터 앞에 앉아 있다면? "이 문서 고쳐줘"라고 말하는 순간, 직접 파일을 열어서 수정하고 저장까지 해요. 내가 할 일이 없어요.

구체적으로 비교하면 이래요:

| | 웹 Claude / ChatGPT | Claude Code |
|---|---|---|
| 파일 읽기 | 내가 복사해서 붙여넣어야 함 | 직접 읽음 |
| 파일 생성 | 내가 받아서 저장해야 함 | 직접 만들어줌 |
| 파일 수정 | 내가 직접 적용해야 함 | 직접 수정해줌 |
| 터미널 명령어 | 알려주기만 함 | 직접 실행할 수 있음 |

1장에서 Claude에게 "파일로 저장해줘"라고 했을 때, 실제로 파일이 생겼죠? 내가 내용을 복사해서 메모장에 붙여넣지 않았는데도요. **이게 Claude Code의 핵심**이에요.

> **NOTE:** Claude Code는 파일을 직접 만들고, 읽고, 수정하고, 삭제할 수 있어요. 이 능력 덕분에 반복 작업을 자동화할 수 있어요. 이 책 전체가 이 능력을 활용하는 방법이에요.

---

## 핵심 개념 3가지

Claude Code를 제대로 쓰려면 딱 3가지만 이해하면 돼요. 이것만 알아도 이 책의 모든 내용을 막힘 없이 따라할 수 있어요.

1. **컨텍스트** — Claude Code는 현재 폴더의 파일들을 자동으로 읽어요
2. **CLAUDE.md** — 프로젝트 설명서. 이걸 쓰면 매번 같은 설명을 반복하지 않아도 돼요
3. **커맨드(스킬)** — 자주 하는 작업을 명령어로 저장해서 한 번에 실행해요

하나씩 살펴볼게요.

---

## 컨텍스트: Claude는 내 폴더를 보고 있어요

Claude Code를 터미널에서 실행하면, 클로드는 **현재 폴더에 어떤 파일이 있는지 자동으로 파악**해요.

예를 들어, `my-resume-pipeline` 폴더에서 `claude`를 실행하면, 클로드는 그 폴더 안의 파일들을 보면서 대화해요. "내 자소서 검토해줘"라고 하면, 폴더 안에 있는 자소서 파일을 찾아서 읽고 검토해요.

이걸 **컨텍스트(context)**라고 해요. 컨텍스트는 "현재 상황 정보"예요. Claude Code가 폴더를 보고 있기 때문에, 내가 일일이 "파일 내용은 이래요..."라고 설명하지 않아도 돼요.

> **TIP:** Claude Code는 항상 터미널의 **현재 폴더**를 기준으로 작동해요. 그래서 `my-resume-pipeline` 폴더에서 실행해야 그 안의 파일들을 볼 수 있어요. 다른 폴더에서 실행하면 내 자소서 파일을 찾지 못해요.

---

## CLAUDE.md: 프로젝트의 "기억"을 만들어요

컨텍스트 덕분에 폴더 안 파일은 자동으로 읽어줘요. 그런데 한 가지 문제가 있어요.

Claude Code는 대화가 끝나면 기억을 잃어요. 오늘 "나는 컴퓨터공학 전공 4학년이고, 카카오에 백엔드 개발자로 지원하고 싶어"라고 했어도, 내일 새 대화를 시작하면 다시 처음부터 설명해야 해요.

이 문제를 해결하는 게 **CLAUDE.md** 파일이에요.

### CLAUDE.md가 뭐예요?

CLAUDE.md는 프로젝트 루트 폴더에 놓아두는 특별한 파일이에요. Claude Code가 시작할 때 **자동으로 읽는** 설명서예요.

여기에 "나는 누구고, 이 프로젝트는 뭐고, 어떤 방식으로 작업하고 싶어요"를 한 번만 써두면 돼요. 그러면 Claude Code는 매번 새 대화를 시작할 때마다 이걸 읽고 "아, 이 프로젝트는 이런 거구나"를 알게 돼요.

책상에 붙여두는 포스트잇 같은 거예요. 내가 매번 말하지 않아도, 클로드가 항상 볼 수 있는 곳에 붙어있는 메모.

### 직접 만들어봐요

Claude Code를 실행하고 이렇게 요청해봐요:

```bash
claude
```

그런 다음 프롬프트에 이렇게 입력해요:

```
CLAUDE.md 파일을 만들어줘. 이 프로젝트는 자소서 작성 파이프라인이야.
나는 [전공] 전공 4학년이고, [회사] [직무]에 지원하고 싶어.
```

대괄호 안은 내 상황에 맞게 바꿔요. 예를 들면:

```
CLAUDE.md 파일을 만들어줘. 이 프로젝트는 자소서 작성 파이프라인이야.
나는 경영학 전공 4학년이고, 카카오 마케팅 직무에 지원하고 싶어.
```

Claude가 CLAUDE.md 파일을 만들어줄 거예요. 내용은 대략 이런 모양이에요:

```markdown
# 자소서 파이프라인 프로젝트

## 프로젝트 소개
이 프로젝트는 맞춤형 자소서 작성을 자동화하는 파이프라인이에요.

## 지원자 정보
- 전공: 경영학 (4학년)
- 목표 회사: 카카오
- 지원 직무: 마케팅

## 작업 방식
- 채용 공고를 분석해서 핵심 키워드를 추출해요
- 추출한 키워드를 바탕으로 맞춤형 자소서를 생성해요
```

### 실험해봐요

CLAUDE.md를 만들었으면, 새 대화를 시작해서 이렇게 물어봐요:

```
내 전공이 뭐야?
```

Claude가 "경영학이에요"라고 대답해요. 내가 방금 말하지 않았는데도요. CLAUDE.md를 읽었으니까요.

이제 CLAUDE.md를 지워보고 같은 질문을 해봐요. Claude가 "저는 사용자분의 전공을 모르는데요, 알려주실 수 있을까요?"라고 할 거예요.

이 차이가 CLAUDE.md의 힘이에요.

> **TIP:** CLAUDE.md는 프로젝트의 "영구 기억"이에요. 한 번 잘 적어두면 Claude Code를 몇 번을 새로 시작해도 항상 내 상황을 기억해요. 나중에 지원하는 회사가 바뀌면 CLAUDE.md만 업데이트하면 돼요.

> **WARNING:** CLAUDE.md에 개인 정보(주민번호, 비밀번호 등)는 절대 적지 마세요. 이 파일은 일반 텍스트 파일이라 보안 처리가 되지 않아요.

## 슬래시 커맨드: 자주 하는 일을 명령어로 저장해요

세 번째 개념이에요. 이게 이 책의 핵심이에요.

자소서 작업을 하다 보면 같은 일을 반복하게 돼요. 채용 공고 분석, 자소서 생성, 검토... 매번 긴 지시사항을 타이핑하는 건 번거로워요.

**슬래시 커맨드**는 이 반복 작업을 명령어 하나로 저장하는 기능이에요.

### 어떻게 작동해요?

`.claude/commands/` 폴더에 `.md` 파일을 만들면, Claude Code에서 `/파일이름` 형태의 명령어로 실행할 수 있어요.

예를 들어, `.claude/commands/hello.md` 파일을 만들면 Claude Code에서 `/hello`라고 입력했을 때 그 파일의 내용대로 Claude가 동작해요.

### 직접 만들어봐요: /hello 커맨드

Claude Code에서 이렇게 요청해요:

```
.claude/commands/hello.md 파일을 만들어줘.
내용은 이렇게:
"인사를 해주세요. 사용자 이름: $ARGUMENTS
짧고 친근하게 인사하고, 오늘 자소서 작성 준비가 됐는지 물어봐주세요."
```

또는 직접 만들어도 돼요. 터미널에서:

**Mac / Linux:**
```bash
mkdir -p .claude/commands
```

**Windows (PowerShell):**
```bash
New-Item -ItemType Directory -Force -Path .claude\commands
```

그런 다음 `.claude/commands/hello.md` 파일을 만들고 이렇게 적어요:

```markdown
인사를 해주세요. 사용자 이름: $ARGUMENTS
짧고 친근하게 인사하고, 오늘 자소서 작성 준비가 됐는지 물어봐주세요.
```

이제 Claude Code를 실행하고 `/hello 홍길동`이라고 입력해봐요.

Claude가 이렇게 응답해요:

```
홍길동님, 안녕하세요! 반갑습니다 😊
오늘 자소서 작성 준비는 어떠세요? 같이 멋진 자소서 만들어봐요!
```

<!-- IMAGE: screenshot-ch02-slash-command.png
[스크린샷 필요] `/hello 홍길동` 실행 화면.
- Claude Code 프롬프트에 `/hello 홍길동` 입력
- Claude가 인사말을 응답하는 화면

`/hello` 뒤에 입력한 `홍길동`이 `$ARGUMENTS` 자리에 들어갔어요.

### $ARGUMENTS가 뭐예요?

`$ARGUMENTS`는 커맨드를 실행할 때 뒤에 붙인 텍스트가 자동으로 들어가는 자리예요.

```
/hello 홍길동
       ^^^^^^
       이게 $ARGUMENTS로 들어가요
```

나중에 만들 `/collect-jd` 커맨드에서는 이런 식으로 써요:

```
/collect-jd kakao-marketing-jd.txt
            ^^^^^^^^^^^^^^^^^^^^^^
            분석할 채용공고 파일 이름이 $ARGUMENTS로 들어가요
```

> **NOTE:** 이 책에서는 Claude Code의 슬래시 커맨드를 **"스킬(skill)"**이라고 부를게요. Claude Code 공식 문서에서는 "커스텀 슬래시 커맨드" 또는 "커맨드"라고 해요. 같은 거예요. 이 책에서 "스킬을 만든다"고 하면, `.claude/commands/` 폴더에 `.md` 파일을 만드는 거라고 이해하면 돼요.

## 이 책에서 만들 4개의 스킬

이제 이 책의 전체 그림을 볼게요. 3장부터 6장까지 스킬을 하나씩 만들어요.

| 스킬 | 하는 일 |
|------|---------|
| `/collect-jd` | 채용 공고(JD)를 분석해서 핵심 정보를 뽑아내요 |
| `/generate-resume` | 분석 결과 + 내 경력으로 맞춤형 자소서를 생성해요 |
| `/review-resume` | 작성된 자소서를 채용 담당자 관점에서 검토해요 |
| `/export-pdf` | 완성된 자소서를 PDF로 내보내요 |

이 4개의 스킬이 하나의 파이프라인을 이뤄요:

```
채용공고 파일
    ↓
/collect-jd
    ↓
분석 결과 (JSON)
    ↓
/generate-resume
    ↓
자소서 초안
    ↓
/review-resume
    ↓
검토 완료된 자소서
    ↓
/export-pdf
    ↓
최종 PDF
```

- /generate-resume: JSON + 경력기술서 → 맞춤형 자소서

-->

각 스킬은 `.claude/commands/` 폴더의 마크다운 파일 하나로 구현돼요. 복잡한 코딩 없이, 마크다운 파일에 지시사항을 적는 것만으로요.

## 지금 내 프로젝트 상태

이번 장을 마치면 프로젝트가 이런 모양이에요:

```
my-resume-pipeline/
  CLAUDE.md                     ← 프로젝트 설명서 (내 정보 포함)
  resume-v1.md                  ← 1장에서 만든 자소서
  .claude/
    commands/
      hello.md                  ← 연습용 커맨드 (/hello)
```

아직 실제 스킬은 없어요. 하지만 중요한 기반을 갖췄어요:

- **CLAUDE.md** 덕분에 Claude가 내 상황을 항상 기억해요
- **스킬**이 어떻게 작동하는지 이해했어요
- 이제 진짜 스킬을 만들 준비가 됐어요

## 정리: 3가지를 기억해요

이번 장에서 배운 3가지 개념을 정리해요:

**1. 컨텍스트**
Claude Code는 현재 폴더의 파일을 자동으로 인식해요. 덕분에 파일 내용을 일일이 복사해서 붙여넣지 않아도 돼요.

**2. CLAUDE.md**
프로젝트 루트에 있는 특별한 파일이에요. Claude Code가 시작할 때 자동으로 읽어요. 내 정보와 프로젝트 정보를 여기 적어두면 매번 설명하지 않아도 돼요.

**3. 슬래시 커맨드(스킬)**
`.claude/commands/` 폴더에 `.md` 파일을 만들면 `/파일이름`으로 실행할 수 있어요. 반복 작업을 명령어 하나로 줄여줘요.

이 3가지가 다음 장부터 만들 자소서 파이프라인의 기반이에요. 이 개념들이 낯설게 느껴져도 괜찮아요. 3장부터 실제로 만들면서 손에 익힐 거예요.

다음 장에서는 첫 번째 진짜 스킬, `/collect-jd`를 만들어요. 채용 공고를 붙여넣으면 핵심 요구사항을 자동으로 뽑아주는 스킬이에요.

---
