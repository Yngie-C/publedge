# 부록 B: 프롬프트 템플릿 모음

책을 따라 포트폴리오를 만들면서 쓴 대화들, 나중에 다시 찾으려면 챕터를 처음부터 뒤져야 했어요. 이 부록은 그 수고를 줄여주는 "프롬프트 치트시트"예요.

본문에서 실제로 쓴 대화 프롬프트를 챕터 순서대로 정리했어요. 처음 포트폴리오를 만들 때뿐 아니라, 나중에 내용을 업데이트하거나 새로 처음부터 시작할 때도 여기서 바로 복사해서 쓸 수 있어요.

## 이 부록을 잘 쓰는 방법

두 가지만 기억해두세요.

**첫째, 템플릿은 시작점이에요.** 그대로 붙여 넣는 것보다 `[내 이름]`, `[한 줄 소개]` 같이 꺾쇠로 표시된 자리를 내 정보로 채우고, 나머지 문구도 상황에 맞게 조금씩 고쳐 쓰는 게 결과가 더 좋아요. Claude는 구체적일수록 정확하게 응답해요.

**둘째, 이 프롬프트들은 Claude Code 대화창이 이미 열린 상태에서 쓰는 거예요.** `my-portfolio` 폴더에서 `claude`를 실행한 뒤, 아래 프롬프트를 대화창에 붙여 넣으면 돼요. 별도 설치나 설정은 필요 없어요.

> **TIP:** 프롬프트를 통째로 붙여 넣기보다, 이름·색·한 줄 소개만 바꾸고 나머지 흐름은 참고용으로 보는 편이 결과가 더 좋아요. 지시가 자연스러울수록 Claude도 자연스럽게 응답해요.

---

## 1. 명함 페이지 (Ch1)

빈 `my-portfolio` 폴더에서 이름과 사진이 담긴 명함 페이지 하나를 만드는 프롬프트예요. Claude Code를 처음 켠 직후에 쓰는 첫 번째 대화예요.

### 스텝 1 — 명함 뼈대 만들기

```text
제 이름은 [내 이름]이고, 한 줄 소개는 "[한 줄 소개]"예요.
이걸 보여주는 HTML 파일 하나 만들어 주세요. 파일 이름은 index.html로 해주세요.

디자인 조건:
- 전체 내용은 화면 가운데 정렬
- 최대 너비 560px인 흰 카드 한 장
- 이름은 굵게, 한 줄 소개는 한 단계 작게
- 배경은 연한 회색(#f5f5f5)
- 별도 라이브러리 없이 인라인 CSS로 완결되게
```

### 스텝 2 — 사진 추가하기

사진 파일(`me.jpg`)을 `my-portfolio` 폴더에 넣은 뒤 아래 프롬프트를 입력해요.

```text
지금 폴더에 me.jpg라는 제 사진을 넣어뒀어요.
index.html 카드 왼쪽에 동그랗게(원형으로) 보이도록 넣어주세요.
사진 크기는 100px 정도, 텍스트와 가로로 나란히 배치해 주세요.
```

**바꿔 쓸 부분**

- 사진이 없다면 스텝 2 대신 "사진 대신 내 이니셜 두 글자를 원형 안에 큼지막하게 넣어줘. 배경색은 올리브 계열로"로 바꿔요.
- 한 줄 소개가 길면 "두 줄까지 허용하고 글자 크기를 0.95rem으로 해줘"를 덧붙여요.
- 카드를 가운데가 아니라 왼쪽 위에서 시작하게 하고 싶으면 "card를 중앙 정렬 대신 페이지 왼쪽 상단 기준으로 배치해줘"로 요청해요.

## 2. 프로젝트 카드 그리드 (Ch2)

명함 아래에 프로젝트를 소개하는 카드 3장을 가로 그리드로 추가하는 프롬프트예요. 기존 `index.html`을 수정하는 거예요.

```text
index.html 아래에 "프로젝트" 섹션을 추가해 주세요.
카드 3장을 가로 그리드로 배치하고, 각 카드는 제목·설명·"자세히 보기" 링크를 담아요.
카드 링크는 새 탭에서 열리게 해주세요 (target="_blank").
명함 카드는 그대로 두고, 명함 아래에만 추가해 주세요. 새 파일은 만들지 말아요.

프로젝트 1: [제목] / [한 줄 설명] / [링크]
프로젝트 2: [제목] / [한 줄 설명] / [링크]
프로젝트 3: [제목] / [한 줄 설명] / [링크]
```

**바꿔 쓸 부분**

- 카드를 4장으로 늘리고 싶으면 프로젝트 줄을 하나 더 추가하고 "카드 4장을 2열 그리드로"라고 바꿔요.
- 링크 텍스트를 "자세히 보기" 대신 "GitHub 보기"나 "결과물 보기"로 바꿀 수 있어요.
- 섹션 제목이 "프로젝트"보다 다른 표현이 좋으면 "섹션 제목은 '내가 만든 것들'로 해줘"처럼 명시해요.

> **WARNING:** "카드 페이지를 새로 만들어줘"라고 쓰면 Claude가 `index.html` 대신 `projects.html` 같은 엉뚱한 파일을 만들 수 있어요. 꼭 "기존 index.html 아래쪽에 추가해줘"라고 명시하세요. 새 파일이 생겼다면 부록 A 케이스 1을 참고해요.

## 3. 감성 업그레이드 (Ch3)

같은 구조에 내 취향의 색·폰트·여백을 입히는 프롬프트예요. 레퍼런스 이미지를 첨부하면 말로 설명하기 어려운 느낌도 전달할 수 있어요. 두 스텝으로 나눠 진행해요.

### 스텝 1 — 취향 키워드 전달

```text
지금 포트폴리오 톤이 조금 딱딱하게 느껴져요.
전체적으로 더 부드럽고 모던한 느낌으로 바꿔 주세요.
구조나 내용은 그대로 두고, 색·폰트·여백만 조정해 주세요.

조건:
- CSS 변수(:root)로 색을 정의해 주세요 (--bg, --surface, --text, --accent)
- 여백은 rem 단위로 통일
- 배경은 크림 또는 올리브 계열, Noto Sans KR 폰트
- 카드 그림자는 rgba(0,0,0,0.06) 이하로 가볍게
```

### 스텝 2 — 레퍼런스 이미지 첨부

핀터레스트나 Behance에서 마음에 드는 포트폴리오 화면을 캡처해서 대화창에 드래그&드롭으로 올린 뒤, 아래 프롬프트를 함께 입력해요.

```text
첨부한 이미지의 색 팔레트와 여백 감각을 참고해서
내 포트폴리오를 이 톤에 가깝게 맞춰주세요.
폰트는 시스템 폰트 스택을 유지하고, 색과 여백만 바꿔요.
콘텐츠 구조(명함, 카드 그리드)는 건드리지 말아요.
```

> **TIP:** 레퍼런스 이미지는 2~3장을 함께 보내면 효과가 좋아요. 한 장에서 "색감"을, 다른 한 장에서 "여백 리듬"을 참고하게 되면 특정 이미지를 너무 가깝게 따라가는 일 없이 자연스럽게 평균화돼요.

**바꿔 쓸 부분**

- 파스텔 톤을 원하면 "--accent는 연한 핑크(#f9a8d4), --bg는 아이보리(#fefce8)"처럼 직접 색 코드를 지정해요.
- 폰트를 Noto Sans KR 대신 시스템 폰트만 쓰고 싶으면 "Google Fonts 링크는 빼고 시스템 폰트 스택만 유지해줘"로 바꿔요.
- 카드 간 여백이 좁다 싶으면 결과물 확인 후 "카드 그리드 gap을 한 단계 넓혀줘, 다른 건 그대로"로 이어서 요청해요.

## 4. 반응형 + 다크모드 토글 (Ch4)

모바일에서도 잘 보이게 하고, 오른쪽 상단에 다크모드 토글 버튼을 다는 프롬프트예요. 두 대화로 나눠서 진행해요.

### 스텝 1 — 모바일 반응형

```text
지금 포트폴리오를 핸드폰(너비 400px 정도)에서 보면 카드 3장이 너무 좁게 붙어 있어요.
화면 너비가 768px보다 작으면 카드를 세로로 1장씩 쌓이게 바꿔 주세요.
명함 카드도 사진-텍스트가 가로가 아니라 세로로 쌓이게 해주세요.
여백도 모바일에서는 조금 줄여주세요. media query로요.
기존 데스크톱 스타일은 건드리지 말아요.
```

### 스텝 2 — 다크모드 토글

```text
오른쪽 상단에 다크모드 토글 버튼을 추가해 주세요.

조건:
- [data-theme="dark"] 속성으로 다크 팔레트를 전환
- 버튼 클릭 시 data-theme 토글, localStorage에 상태 저장
- 페이지를 새로고침해도 이전 선택이 유지
- 다크 팔레트는 현재 :root 변수 기준으로 --bg, --surface를 어둡게, --text를 밝게만 정의
- JS는 최소한으로, 프레임워크 없이
```

**바꿔 쓸 부분**

- 브레이크포인트를 768px이 아닌 다른 값으로 하고 싶으면 "768px 대신 480px 기준으로 해줘"로 바꿔요.
- 다크 팔레트 색을 직접 지정하고 싶으면 "--bg-dark: #1a1a2e, --surface-dark: #16213e" 같이 원하는 값을 붙여 넣어요.
- 토글 버튼 아이콘을 문자 대신 이모지로 쓰고 싶으면 "아이콘은 🌙/☀️ 이모지로 해줘"를 덧붙여요.

> **NOTE:** 다크모드 팔레트를 한 번에 정하기 어려우면 "라이트 쪽 배경 색을 단순히 어두운 계열로 반전만 시켜서 임시로 써봐"처럼 점진적으로 접근하는 것도 좋아요. 임시 팔레트를 먼저 보고 마음에 드는 방향으로 조금씩 수정하면 훨씬 수월해요.

## 보너스: 자주 쓰는 수정 패턴

포트폴리오를 완성하고 나서도 "이것만 살짝 바꾸고 싶다"는 순간이 와요. 그럴 때 바로 쓸 수 있는 짧은 프롬프트 3개예요.

### 1. 여백만 살짝 넓히기

```text
전체적으로 카드 사이 간격(gap)과 카드 내부 여백(padding)을
각각 한 단계씩 넓혀줘. 다른 스타일은 건드리지 말고.
```

### 2. 카드에 호버 효과 추가

```text
프로젝트 카드에 마우스를 올리면 살짝 떠오르는 효과를 넣어줘.
box-shadow를 강하게, translateY(-4px) 정도. transition 포함해줘.
명함 카드엔 적용하지 말고, 프로젝트 카드에만요.
```

### 3. 글자 크기만 조정

```text
이름(h1)을 한 단계 크게, 한 줄 소개를 한 단계 작게 조정해줘.
다른 텍스트 크기는 그대로.
```

## 프롬프트 잘 쓰는 작은 습관

템플릿을 복사해서 쓰다 보면 자연스럽게 느끼게 되는 패턴이 있어요. 처음부터 의식하면 결과물이 달라지는 세 가지 습관을 정리했어요.

### 1. 수치를 구체적으로 넣기

"조금 크게"보다 "1.2rem에서 1.5rem으로"처럼 숫자를 직접 주면 결과가 훨씬 예측 가능해요. Claude가 "조금"이라는 감각을 스스로 해석해야 할 때와 수치가 주어졌을 때, 후자가 원하는 방향에 가깝게 나올 확률이 크게 높아요.

색을 바꿀 때도 마찬가지예요. "더 따뜻하게"보다 "--accent를 #5a6a3e에서 #8b5e3c로 바꿔줘"처럼 말하면 의도를 정확하게 전달할 수 있어요. 색상 코드는 구글에서 원하는 색을 검색하면 바로 나와요.

### 2. 한 번에 하나씩 요청하기

한 프롬프트에 색 변경, 폰트 교체, 여백 조정, 호버 효과를 한꺼번에 넣으면 Claude가 뭉뚱그려 처리하거나 일부를 빠뜨리는 경우가 있어요. "색부터 먼저, 폰트는 다음 대화에서"처럼 작게 나눠 대화 턴을 쓰는 편이 결과물 품질이 더 좋아요.

Ch3에서 취향 루프를 두 번으로 나눈 것도 같은 이유예요. 한 번에 모든 것을 바꾸려 하지 않고, 방향을 잡고 → 확인하고 → 조정하는 흐름이 **Vibe Coding**의 자연스러운 리듬이에요.

### 3. 스크린샷으로 진단 먼저 시키기

결과물이 마음에 안 드는데 뭐가 문제인지 잘 모를 때가 있어요. 그럴 때 "이게 어색해 보여. 고쳐줘"보다 Claude에게 먼저 진단을 맡기는 게 효과적이에요.

브라우저에서 화면을 캡처하고 대화창에 붙여 넣은 뒤 이렇게 물어봐요.

```text
지금 화면이야. 어떤 부분이 어색해 보여?
개선할 부분을 2~3개만 짚어줘.
```

Claude가 문제를 먼저 짚어주면, 거기서 마음에 드는 항목을 골라 "그 중에 카드 간격 문제만 먼저 고쳐줘"라고 이어가면 돼요. 직접 문제를 찾아서 설명하는 것보다 훨씬 빠르게 원하는 방향으로 수렴할 수 있어요.

## 마무리

이 네 가지 템플릿을 조합하면 Ch1부터 Ch4까지 본문에서 만든 포트폴리오를 처음부터 다시 재현할 수 있어요. 나중에 이직 준비로 내용을 갱신할 때, 또는 취향이 바뀌어 디자인을 새로 입히고 싶을 때 여기서 필요한 프롬프트만 꺼내 쓰면 돼요.

부록 C에서는 이 포트폴리오를 더 확장하고 싶은 분들을 위한 다음 선택지를 안내해요. 자소서 책으로 넘어가는 경로, **Vibe Coding**을 더 깊이 탐험하는 세 가지 아이디어, 그리고 커스텀 도메인 연결까지 이어집니다.
