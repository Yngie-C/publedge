# 첫 결과물: "안녕 세상, 제 이름은 ___입니다"

Ch0에서 `claude`를 처음 띄웠던 그 빈 폴더, 기억하시죠? 터미널에 `claude`를 쳤을 때 아무 일도 안 일어난 것 같았지만, 사실 도구는 이미 준비됐어요. 이번 챕터에서는 그 폴더에 실제로 눈에 보이는 첫 결과물을 만들어볼 거예요. 대화 두 번이면 충분해요.

## 이번 챕터의 목표

- 내 이름과 한 줄 소개를 Claude에게 알려주고 HTML 파일을 만들어요
- Claude가 생성한 `index.html`을 더블클릭으로 브라우저에서 열어서 확인해요
- `me.jpg` 사진을 폴더에 넣고 명함 페이지를 완성해요

이게 끝이에요. 코드를 직접 쓸 필요 없어요. 대화만 하면 돼요.

---

## Claude Code 다시 실행하기

Ch0 마지막에 Claude Code를 껐다면 터미널을 열고 아래 명령어를 입력해요.

```bash
cd my-portfolio
claude
```

`cd my-portfolio`는 아까 만들었던 폴더 안으로 들어가는 명령어예요. 그 다음 `claude`를 치면 Claude Code가 다시 켜져요.

화면에 이런 프롬프트가 뜨면 준비 완료예요.

```
╭────────────────────────────────────╮
│ Claude Code                        │
│ /Users/yourname/my-portfolio       │
╰────────────────────────────────────╯

>
```

이미 Claude Code가 켜져 있다면 이 섹션은 건너뛰고 바로 다음으로 넘어가요.

폴더 위치가 헷갈린다면 터미널에서 `pwd`를 입력해보세요. 현재 위치가 출력돼요. `/Users/yourname/my-portfolio` 처럼 `my-portfolio`로 끝나면 맞게 들어와 있는 거예요. 다른 위치가 출력된다면 `cd ~/my-portfolio`를 입력해서 이동할 수 있어요.

---

## 첫 프롬프트 쓰기

`>` 뒤에 아래 예시를 그대로 입력하면 돼요. **이름과 한 줄 소개 부분만 내 것으로 바꿔주세요.**

```markdown
제 이름은 김민지이고, 한 줄 소개는 "데이터로 세상을 더 쉽게 만드는 사람"이에요.
이걸 보여주는 간단한 HTML 파일 하나 만들어 주세요. 파일 이름은 index.html로 해주세요.
```

Enter를 누르면 Claude가 잠깐 생각한 다음 파일을 만들기 시작해요. 보통 30초 안에 끝나요.

한 줄 소개를 어떻게 써야 할지 막막하다면 직업이나 관심사를 담아 자연스럽게 적어보세요. 예를 들면 이런 식이에요.

- "UI가 사람을 움직인다고 믿는 프로덕트 디자이너"
- "코드보다 사람에 더 관심 많은 개발자"
- "사진 찍는 걸 좋아하는 마케터"

한 줄 소개는 나중에 언제든 바꿀 수 있어요. 지금은 일단 뭐든 적고 시작해요.

이 프롬프트가 짧고 단순해 보여도 괜찮아요. Claude는 짧은 지시도 충분히 이해해요. 오히려 너무 길게 쓰면 의도가 흐려질 수 있어요. 핵심 정보만 간결하게 담는 게 바이브 코딩의 좋은 습관이에요.

> **TIP:** 프롬프트 끝에 "파일 이름은 index.html로 해주세요"를 꼭 붙여요. Claude가 엉뚱한 파일 이름을 선택하는 일을 줄일 수 있어요. 만약 의도치 않은 파일이 생겼다면 부록 A의 '새 파일 생성' 항목을 참고하세요.

## Claude의 응답 읽기

Claude가 작업을 마치면 대화창에 이런 식으로 응답이 나타나요.

```
index.html 파일을 생성했어요.

파일에는 다음이 들어 있어요.
- 이름: 김민지
- 한 줄 소개: 데이터로 세상을 더 쉽게 만드는 사람
- 기본적인 스타일 (중앙 정렬, 배경색, 글꼴)

브라우저에서 index.html 파일을 열어서 확인해 보세요.
```

Claude는 파일 내용을 직접 터미널에 보여주기도 하고, 요약만 보여주기도 해요. 어느 쪽이든 실제 파일이 폴더에 저장된 거예요.

응답이 길게 올라가서 화면 밖으로 스크롤됐다면 당황하지 말아요. 내용이 아무리 많이 출력돼도 파일 저장은 이미 완료된 거예요. 중요한 건 대화창 내용이 아니라 `my-portfolio` 폴더 안에 `index.html`이 있느냐예요.

`my-portfolio` 폴더를 열어보면 `index.html` 파일이 들어 있을 거예요. Finder(macOS) 또는 파일 탐색기(Windows)에서 확인할 수 있어요.

---

## 더블클릭으로 열어보기

이제 브라우저로 열어볼 차례예요.

**macOS:** Finder에서 `my-portfolio` 폴더를 열고 `index.html` 파일을 더블클릭해요. Chrome이나 Safari 같은 기본 브라우저에서 파일이 열려요.

**Windows:** 파일 탐색기에서 `my-portfolio` 폴더를 열고 `index.html`을 더블클릭해요. 기본 브라우저가 자동으로 실행돼요.

화면에 내 이름과 한 줄 소개가 딱 찍혀 있어요.

"이게 내가 만든 첫 사이트예요!"

아직 사진도 없고 디자인도 단순하지만, 코드를 한 줄도 직접 짜지 않고 대화 한 번으로 만든 페이지예요. 이게 바이브 코딩의 시작이에요.

브라우저 주소창을 보면 `file:///Users/yourname/my-portfolio/index.html` 같은 경로가 보일 거예요. 아직 인터넷에 올라간 건 아니에요. 내 컴퓨터에서만 열리는 로컬 파일이에요. 인터넷에 공개하는 건 Ch5에서 할 거예요. 지금은 브라우저로 확인하는 것만으로 충분해요.

이 단계에서 디자인이 마음에 쏙 들지 않아도 괜찮아요. 배경색이 밋밋하거나 글씨 크기가 어색하게 느껴질 수 있어요. 그건 Ch3에서 취향에 맞게 바꿀 거예요. 지금은 이름과 한 줄 소개가 화면에 보이는 것 자체가 목표예요.

> **NOTE:** 브라우저가 자동으로 열리지 않는다면, 브라우저를 직접 실행한 다음 `Cmd + O`(macOS) 또는 `Ctrl + O`(Windows)를 눌러 파일을 직접 열 수도 있어요.

## 사진 추가하기

텍스트만 있는 명함 페이지도 충분하지만, 사진이 있으면 훨씬 더 사람 냄새가 나요. 방문자가 포트폴리오를 봤을 때 "이 사람이구나"라는 느낌을 받게 해주는 게 사진이에요.

먼저 사진 파일을 준비해요. 스마트폰 갤러리에서 괜찮은 사진을 하나 골라서 컴퓨터로 옮겨요. 꼭 전문 증명사진일 필요는 없어요. 얼굴이 잘 보이는 사진이면 충분해요.

사진 파일을 골랐으면 이름을 `me.jpg`로 바꿔요.

- **macOS:** 파일을 우클릭 → "이름 변경"
- **Windows:** 파일을 우클릭 → "이름 바꾸기"

이름을 바꿨으면 `my-portfolio` 폴더 안으로 드래그&드롭해서 넣어요. Finder나 파일 탐색기에서 폴더를 열어두고 파일을 끌어다 놓으면 돼요.

폴더 안에 `me.jpg`가 들어간 걸 확인했으면, Claude에게 두 번째 프롬프트를 입력해요.

```markdown
지금 폴더에 me.jpg라는 제 사진을 넣어뒀어요.
index.html 상단 왼쪽에 동그랗게(원형으로) 보이도록 넣어주세요.
```

Claude가 `index.html`을 열고 사진을 원형으로 보여주는 코드를 추가해줄 거예요. 기존 파일을 덮어쓰는 게 아니라 이미지 태그를 추가하는 방식이라 기존 내용은 그대로 유지돼요.

> **NOTE:** 브라우저에서 새로고침했는데 사진이 안 보인다면 부록 A의 '이미지 경로 문제' 항목을 참고해요. 파일 이름 대소문자나 확장자가 다른 경우가 가장 흔한 원인이에요.

## 최종 명함 페이지 확인하기

Claude가 파일 수정을 마쳤으면 브라우저에서 `F5`(Windows) 또는 `Cmd + R`(macOS)을 눌러 새로고침해요.

이름 옆에 내 사진이 동그란 원형으로 들어간 명함 페이지가 나타나요. 이게 Ch1의 최종 결과물이에요.

사진 크기나 위치가 마음에 안 들면 바로 Claude에게 다시 말해도 돼요. "사진을 조금 더 크게 해주세요"나 "사진을 왼쪽 말고 가운데 위에 놓아주세요"처럼 느낌 그대로 요청하면 Claude가 수정해줘요. 한 번에 완벽하지 않아도 괜찮아요. 바이브 코딩은 대화를 이어가면서 조금씩 다듬어 가는 과정이에요.

<!-- IMAGE: screenshot-ch01-final-card.png
사진이 동그란 원형으로 들어간 완성된 명함 페이지 브라우저 스크린샷.
이름, 원형 사진, 한 줄 소개가 함께 표시된 상태.
-->

아래는 Claude가 만들어준 `index.html`의 실제 모습이에요. 직접 수정하거나 다음 챕터에서 내용을 추가할 때 기준이 되는 파일이에요.

파일을 열어보면 `<style>` 태그 안에 CSS가 들어 있고, `<body>` 안에 이름과 사진, 한 줄 소개가 담겨 있어요. 지금 당장 코드를 이해하지 않아도 괜찮아요. "이런 구조로 되어 있구나" 정도만 눈에 익혀두세요. 다음 챕터에서 Claude가 이 파일 아래에 새로운 섹션을 이어 붙일 거예요.

코드를 보면서 "이게 뭔지 하나도 모르겠는데?"라고 느꼈다면 완전히 정상이에요. 바이브 코딩은 코드를 이해하고 짜는 방식이 아니에요. Claude가 짜준 코드를 눈으로 보고, 결과를 브라우저에서 확인하고, 마음에 안 들면 말로 고쳐달라고 하는 과정이에요. HTML, CSS 문법을 외울 필요가 없어요.

> **WARNING:** `index.html` 파일을 텍스트 편집기로 직접 수정하다가 오류가 나는 경우가 있어요. 내용을 바꾸고 싶을 때는 편집기 대신 Claude에게 말로 요청하는 게 더 안전해요.

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>김민지 포트폴리오</title>
  <style>
    * { box-sizing: border-box; margin: 0; padding: 0; }
    body {
      min-height: 100vh;
      display: flex;
      align-items: center;
      justify-content: center;
      background-color: #f5f5f5;
      font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
    }
    .card {
      background: #ffffff;
      border-radius: 16px;
      box-shadow: 0 4px 24px rgba(0, 0, 0, 0.08);
      padding: 48px 56px;
      display: flex;
      align-items: center;
      gap: 36px;
      max-width: 560px;
      width: 100%;
    }
    .avatar {
      width: 100px;
      height: 100px;
      border-radius: 50%;
      object-fit: cover;
      flex-shrink: 0;
      border: 3px solid #e8e8e8;
    }
    .info h1 {
      font-size: 1.6rem;
      font-weight: 700;
      color: #1a1a1a;
      margin-bottom: 10px;
    }
    .tagline {
      font-size: 1rem;
      color: #555555;
      line-height: 1.6;
    }
  </style>
</head>
<body>
  <div class="card">
    <img class="avatar" src="me.jpg" alt="프로필 사진">
    <div class="info">
      <h1>김민지</h1>
      <p class="tagline">데이터로 세상을 더 쉽게 만드는 사람</p>
    </div>
  </div>
</body>
</html>
```

## 여기까지 왔다면

Ch0에서 빈 폴더만 있었는데, 지금은 내 이름과 사진이 담긴 명함 페이지 하나가 생겼어요.

코드를 외울 필요도 없었고, HTML이 뭔지 몰라도 됐어요. 그냥 Claude에게 원하는 걸 말했더니 만들어줬어요. 이게 바이브 코딩의 핵심이에요.

지금 만든 명함 페이지는 아직 아무도 못 봐요. 내 컴퓨터에서만 열리는 파일이니까요. 하지만 구조는 이미 완성됐어요. 이름, 사진, 한 줄 소개. 포트폴리오의 첫 번째 층이 쌓인 거예요.

이 상태에서 브라우저 탭을 닫지 말고 열어두세요. 다음 챕터에서 파일을 수정하고 새로고침만 하면 변화가 바로 보여요.

Ch0 → Ch1, 딱 두 단계 만에 빈 폴더가 내 얼굴이 들어간 페이지로 바뀌었어요. 이 속도가 바이브 코딩이에요.

지금 이 파일은 나중에 Ch2, Ch3, Ch4를 거치면서 계속 진화할 거예요. 매 챕터가 끝날 때마다 브라우저에서 달라진 모습을 확인할 수 있어요.

다음 챕터에서는 이 명함 아래에 프로젝트 카드 3장을 나란히 놓아볼게요. 내가 해온 일들을 방문자에게 보여주는 섹션이에요.
