# 부록 A: 트러블슈팅 가이드

설치나 실행 중에 문제가 생겼나요? 걱정 마세요. 대부분의 문제는 여기서 해결할 수 있어요. 오류 메시지를 보고 해당 항목을 찾아서 따라 하면 금방 해결될 거예요. 오류가 발생했을 때 당황하지 말고 메시지를 천천히 읽어보는 것만으로도 원인의 절반은 파악할 수 있어요.

---

## 설치 관련 문제

### `npm: command not found`

터미널에서 `npm`을 입력했을 때 이 오류가 나오면 Node.js가 설치되지 않은 거예요. npm은 Node.js에 포함된 패키지 관리자라서, Node.js를 먼저 설치해야 써요.

**해결 방법:**

1. [nodejs.org](https://nodejs.org)에 접속해서 **LTS 버전**을 다운로드하세요. Current 버전보다 LTS(Long Term Support) 버전이 더 안정적이에요.
2. 설치 마법사를 따라 설치를 완료해요. Windows는 `.msi` 설치 파일, macOS는 `.pkg` 파일을 사용해요.
3. **터미널을 완전히 닫고 다시 열어요.** 이 단계를 빠뜨리는 분들이 많은데, 설치 후에는 반드시 터미널을 새로 열어야 PATH 변경 사항이 적용돼요.
4. `node --version`과 `npm --version`을 입력해서 버전 번호가 뜨면 성공이에요.

> **NOTE:** macOS는 Homebrew로도 설치할 수 있어요. 터미널에서 `brew install node`를 실행하면 돼요. Homebrew가 없다면 [brew.sh](https://brew.sh)를 먼저 설치하세요. Homebrew를 이미 쓰고 있다면 이 방법이 더 편리해요.

---

### `EACCES: permission denied`

npm으로 패키지를 전역 설치(`-g` 옵션)할 때 권한 오류가 나는 경우예요. npm의 전역 설치 폴더가 일반 사용자 권한으로는 접근할 수 없는 시스템 폴더에 있기 때문이에요.

**macOS:**

```bash
sudo npm install -g @anthropic-ai/claude-code
```

비밀번호를 입력하라고 하면 Mac 로그인 비밀번호를 입력하세요. 보안상 이유로 입력해도 화면에 아무것도 표시되지 않는 게 정상이에요.

**Windows:**

시작 메뉴에서 **명령 프롬프트**를 검색하고, 우클릭해서 **"관리자 권한으로 실행"**을 선택하세요. 그 다음 설치 명령어를 다시 입력하면 돼요.

> **TIP:** 권한 문제를 근본적으로 해결하고 싶다면 **nvm(Node Version Manager)**을 사용하세요. nvm으로 설치한 Node.js는 사용자 홈 폴더 안에 설치되기 때문에 `sudo` 없이도 전역 패키지를 자유롭게 설치할 수 있어요. macOS/Linux는 [nvm-sh/nvm](https://github.com/nvm-sh/nvm), Windows는 [nvm-windows](https://github.com/coreybutler/nvm-windows)를 참고하세요. 개발을 계속할 예정이라면 nvm 방식을 강력히 추천해요.

---

### `claude: command not found`

`npm install -g`로 Claude Code 설치는 성공했는데 `claude` 명령어가 인식이 안 되는 경우예요. npm 전역 설치 경로가 시스템 PATH에 등록되지 않아서 생기는 문제예요.

**macOS/Linux 확인 방법:**

```bash
npm config get prefix
```

이 명령어를 실행하면 경로가 나와요 (예: `/usr/local` 또는 `/Users/username/.nvm/versions/node/v20.0.0`). 이 경로 뒤에 `/bin`을 붙인 폴더가 PATH에 포함되어 있어야 해요.

PATH 확인:

```bash
echo $PATH
```

출력된 내용에 위 경로가 없으면 `~/.zshrc`(zsh 사용자) 또는 `~/.bash_profile`(bash 사용자) 파일에 다음 줄을 추가하세요.

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

추가 후 터미널을 다시 열거나 아래 명령어로 즉시 반영할 수 있어요.

```bash
source ~/.zshrc
```

**Windows:**

터미널을 완전히 닫고 다시 열어보세요. 그래도 안 되면 Node.js를 공식 사이트에서 다시 설치하면 설치 과정에서 PATH가 자동으로 설정돼요.

```bash
npm install -g @anthropic-ai/claude-code
```

재설치 후에도 안 되면 Windows 검색창에서 **"환경 변수 편집"**을 열고, `Path` 항목에 npm 전역 폴더 경로가 있는지 확인하세요.

### Windows PowerShell 실행 정책 오류

Windows에서 스크립트를 실행하려고 하면 이런 메시지가 나올 수 있어요.

```
이 시스템에서 스크립트를 실행할 수 없습니다. 자세한 내용은
about_Execution_Policies를 참조하십시오.
```

Windows는 기본적으로 외부에서 내려받은 스크립트 실행을 차단해요. Claude Code 설치 과정에서 이 정책에 걸릴 수 있어요.

**해결 방법 1: 실행 정책 변경**

PowerShell을 **관리자 권한으로 실행**하고 다음 명령어를 입력하세요.

```bash
Set-ExecutionPolicy RemoteSigned
```

`Y`를 입력해서 확인하고, PowerShell을 닫았다가 다시 열어요. `RemoteSigned`는 로컬 스크립트는 자유롭게 실행하고, 인터넷에서 내려받은 스크립트는 서명된 것만 허용하는 합리적인 설정이에요.

**해결 방법 2: 명령 프롬프트 사용**

PowerShell 대신 **명령 프롬프트(cmd.exe)**를 사용하면 이 오류 없이 바로 실행돼요. 시작 메뉴에서 `cmd`를 검색해서 열면 돼요. 이 책의 모든 명령어는 cmd에서도 동일하게 작동해요.

### Node.js 버전이 낮을 때

Claude Code는 Node.js 18 이상이 필요해요. 예전에 Node.js를 설치해 두었다면 버전이 낮아서 실행이 안 될 수 있어요.

```bash
node --version
```

v18.0.0 미만이 출력되면 [nodejs.org](https://nodejs.org)에서 최신 LTS 버전을 다시 설치하세요. 기존 버전 위에 덮어쓰기로 설치하면 돼요. 설치 후 터미널을 재시작하고 버전을 다시 확인해 보세요.

## 인증 관련 문제

### `claude auth login` 후 브라우저가 안 열릴 때

`claude auth login`을 실행하면 자동으로 브라우저가 열리면서 로그인 페이지로 이동해야 해요. 그런데 일부 환경(특히 원격 서버, WSL, 일부 Linux 환경)에서는 터미널에서 브라우저를 자동으로 열지 못해요.

**해결 방법:**

터미널 출력을 잘 보면 인증 URL이 함께 표시되어 있어요. 그 URL을 마우스로 드래그해서 전체 선택한 다음, 복사(`Cmd+C` / `Ctrl+C`)해서 브라우저 주소창에 붙여넣기 하면 돼요.

> **NOTE:** URL이 `https://claude.ai/...`로 시작하는 꽤 긴 주소예요. 중간에 끊기지 않도록 전체를 복사해야 정상적으로 인증이 돼요.

---

### "Not authenticated" 오류

Claude Code를 실행했을 때 인증이 되지 않았다는 오류가 나오면 로그인 세션이 만료됐거나, 처음부터 로그인이 되지 않은 거예요.

```bash
claude auth login
```

이미 로그인된 상태에서도 이 오류가 반복된다면, 기존 세션을 지우고 다시 로그인해 보세요.

```bash
claude auth logout
claude auth login
```

로그아웃 후 다시 로그인하면 대부분 해결돼요.

---

### Pro/Max 구독이 없을 때

Claude Code는 Claude.ai의 **Pro 또는 Max 플랜** 구독이 필요해요. 무료 플랜으로는 Claude Code 기능을 사용할 수 없어요. 로그인은 되는데 실행하면 구독 관련 오류가 나오는 경우가 이에 해당해요.

**확인 방법:**

[claude.ai/settings](https://claude.ai/settings)에 접속해서 현재 플랜을 확인하세요. 무료 플랜이라면 Pro 플랜으로 업그레이드해야 해요.

> **NOTE:** Claude Code는 API 키 방식이 아닌 **OAuth 로그인 방식**을 사용해요. `claude auth login`을 실행하면 브라우저에서 claude.ai 계정으로 로그인하는 방식이에요. 인터넷에서 찾은 가이드 중 `ANTHROPIC_API_KEY` 환경 변수를 설정하라는 안내가 있어도, 이 책의 파이프라인에서는 그 방법을 사용하지 않아요. API 키 설정 없이 `claude auth login`만으로 충분해요.

---

### 네트워크 오류 / 타임아웃

인터넷 연결 문제나 특수한 네트워크 환경 때문에 인증이나 AI 응답 요청이 실패할 수 있어요. 오류 메시지에 `ECONNREFUSED`, `ETIMEDOUT`, `network error` 같은 단어가 포함되어 있다면 네트워크 문제일 가능성이 높아요.

**순서대로 확인해 보세요:**

1. 브라우저에서 다른 사이트가 정상적으로 열리는지 기본 인터넷 연결을 먼저 확인해요.
2. **VPN**을 사용 중이라면 VPN을 끄고 다시 시도해 보세요. 일부 VPN은 Anthropic 서버와의 통신을 차단할 수 있어요.
3. 회사나 학교 네트워크를 사용 중이라면 IT 부서의 **방화벽이나 프록시**가 외부 API 요청을 차단하고 있을 수 있어요. 이 경우 개인 핫스팟이나 집 와이파이에서 시도해 보세요.

## 실행 관련 문제

### "파일을 찾을 수 없습니다"

Claude Code에게 특정 파일을 읽어달라고 했을 때 파일을 찾지 못하는 경우예요. Claude Code는 현재 작업 폴더를 기준으로 파일을 찾기 때문에, 올바른 폴더에서 실행하는 게 중요해요.

**확인 방법:**

현재 어느 폴더에 있는지 먼저 확인하세요.

```bash
# macOS/Linux
pwd

# Windows
cd
```

`my-resume-pipeline` 폴더 안에 있어야 해요. 다른 폴더에 있다면 이동하세요.

```bash
cd ~/Desktop/my-resume-pipeline
```

파일 경로를 직접 지정할 때는 오타 여부를 확인하고, macOS/Linux에서는 파일 이름의 대소문자도 정확히 맞춰야 해요 (대소문자를 구분해요). Windows는 대소문자를 구분하지 않아서 좀 더 유연해요.

---

### Claude가 엉뚱한 답을 할 때

프롬프트를 입력했는데 원하는 방향과 전혀 다른 결과가 나오거나, 파이프라인과 관계없는 일반적인 답변만 하는 경우예요.

**순서대로 확인해 보세요:**

1. `CLAUDE.md` 파일 내용을 다시 읽어봐요. 파이프라인의 목적, 역할, 파일 구조가 명확하게 적혀 있어야 Claude가 문맥을 제대로 이해할 수 있어요. 내용이 너무 짧거나 모호하다면 보충해 주세요.
2. 요청을 더 구체적으로 작성해 보세요. "자소서 써줘"처럼 짧은 요청보다는 "data/jd/kakao.md의 채용 공고를 보고, data/experience.md에 있는 내 경험을 바탕으로 자소서 초안 4개 항목을 작성해줘"처럼 파일 경로와 원하는 결과를 명시하는 게 훨씬 효과적이에요.
3. 대화가 너무 길어졌을 수 있어요. 여러 번 주고받다 보면 이전 내용이 문맥을 복잡하게 만들 수 있어요. `claude`를 종료하고 다시 실행해서 새 대화로 시작해 보세요.

> **NOTE:** AI 응답은 매번 조금씩 달라요. 똑같은 입력을 줘도 매번 다른 표현으로 답변이 나오는 게 정상이에요. 핵심 구조 (JSON 형식, 자소서 항목 개수, 파일 저장 위치 등)가 비슷하다면 잘 작동하고 있는 거예요.

---

### 슬래시 커맨드가 작동하지 않을 때

`/collect-jd`나 `/generate-resume` 같은 슬래시 커맨드를 입력했는데 "알 수 없는 명령어"라고 나오거나, 아무 반응 없이 무시되는 경우예요.

**확인할 것들:**

1. `.claude/commands/` 폴더가 `my-resume-pipeline` 폴더 안에 있는지 확인해요.

```bash
# macOS/Linux
ls .claude/commands/

# Windows
dir .claude\commands\
```

파일 목록이 나와야 해요. 폴더가 없거나 비어 있다면 책의 해당 챕터를 다시 따라해서 파일을 만들어야 해요.

2. 파일 확장자가 반드시 `.md`여야 해요. `.txt`나 확장자 없이 저장된 경우 슬래시 커맨드로 인식되지 않아요. Windows는 파일 탐색기에서 확장자를 숨기는 경우가 있어서, "폴더 옵션 → 알려진 파일 형식의 파일 확장명 숨기기"를 해제하고 확인해 보세요.

3. **`my-resume-pipeline` 폴더 안에서 `claude`를 실행했는지 확인해요.** 슬래시 커맨드는 현재 폴더를 기준으로 `.claude/commands/` 경로를 찾아요. 다른 폴더에서 claude를 실행하면 커맨드 파일을 찾지 못해요.

```bash
cd ~/Desktop/my-resume-pipeline
claude
```

### 응답 중간에 끊기거나 느릴 때

Claude Code가 답변을 생성하다가 갑자기 멈추거나, 응답이 평소보다 매우 느린 경우예요.

**원인과 대처 방법:**

- **Anthropic 서버 상태 문제:** 가끔 서버가 혼잡하거나 점검 중일 수 있어요. [status.anthropic.com](https://status.anthropic.com)에서 서비스 상태를 확인할 수 있어요.
- **요청한 작업이 너무 많은 경우:** 한 번에 너무 많은 파일을 분석하거나 너무 긴 텍스트를 처리하면 느려질 수 있어요. 작업을 나눠서 요청해 보세요.
- **네트워크 불안정:** 와이파이 신호가 약하거나 인터넷이 불안정하면 응답이 중간에 끊길 수 있어요. 유선 연결이나 안정적인 네트워크에서 다시 시도해 보세요.

> **TIP:** Claude가 응답을 멈춘 것처럼 보여도 실제로 긴 작업을 처리 중일 수 있어요. `...` 표시가 계속 깜빡인다면 기다려 보세요. 완전히 멈춘 것 같으면 `Ctrl+C`로 중단하고 더 짧은 요청으로 다시 시도하세요.

## PDF 관련 문제

### `md-to-pdf` 설치 실패

```bash
npm install -g md-to-pdf
```

이 명령어 실행 중 오류가 나는 경우예요.

**권한 오류인 경우:**

```bash
# macOS
sudo npm install -g md-to-pdf

# Windows: 관리자 권한으로 명령 프롬프트 실행 후
npm install -g md-to-pdf
```

**Node.js 버전 문제인 경우:**

md-to-pdf도 Node.js 18 이상이 필요해요. 앞의 "Node.js 버전이 낮을 때" 항목을 참고해서 먼저 Node.js를 업그레이드하세요.

**설치 중 `gyp` 또는 빌드 도구 오류가 나는 경우:**

macOS에서 Xcode Command Line Tools가 없어서 생길 수 있어요.

```bash
xcode-select --install
```

위 명령어로 설치 후 다시 시도해 보세요.

### Chromium 다운로드 실패

md-to-pdf는 내부적으로 Chromium(크롬 브라우저 엔진)을 사용해서 마크다운을 PDF로 변환해요. 처음 설치하거나 처음 실행할 때 Chromium을 자동으로 다운로드하는데, 이 과정이 실패하는 경우가 있어요.

**원인과 해결 방법:**

- 인터넷 연결이 불안정한 경우: 잠시 기다렸다가 다시 시도해요.
- 회사/학교 네트워크 방화벽이 외부 다운로드를 차단한 경우: 개인 네트워크(집 와이파이, 핫스팟)에서 한 번만 설치해요.
- 디스크 용량이 부족한 경우: Chromium은 약 300~400MB가 필요해요. 저장 공간을 확인해 보세요.

> **TIP:** md-to-pdf 없이도 PDF를 만들 수 있어요. 마크다운 파일을 VS Code나 브라우저에서 열고 인쇄 기능(`Cmd+P` / `Ctrl+P`)을 사용하세요. 인쇄 대화상자에서 프린터 대신 **"PDF로 저장"**을 선택하면 돼요. 급할 때 유용한 대안이에요.

### PDF에서 한글이 깨질 때

PDF를 열었을 때 한글이 네모나 물음표로 표시되는 경우예요. 시스템에 한글을 표시할 수 있는 폰트가 없어서 생기는 문제예요.

**macOS:**

macOS는 기본적으로 애플고딕, 나눔고딕 등 한글 폰트가 설치되어 있어서 대부분 문제가 없어요. 그래도 깨진다면 `md-to-pdf`를 최신 버전으로 업데이트해 보세요.

```bash
npm update -g md-to-pdf
```

**Windows:**

Windows에서는 한글 폰트가 설치되어 있지 않을 수 있어요. **나눔고딕** 폰트를 설치하면 해결되는 경우가 많아요.

1. 네이버에서 "나눔폰트 다운로드"를 검색하거나 [hangeul.naver.com](https://hangeul.naver.com)에 접속해요.
2. 나눔고딕 폰트를 다운로드하고 설치해요. (파일을 우클릭 → "모든 사용자용으로 설치")
3. 터미널을 재시작하고 md-to-pdf를 다시 실행해요.

### PDF 파일이 생성되지 않거나 크기가 0일 때

변환 명령어를 실행했는데 파일이 생기지 않거나, 생겼어도 크기가 0바이트여서 열리지 않는 경우예요.

**확인 방법:**

```bash
# macOS/Linux
ls -la output/pdf/

# Windows
dir output\pdf\
```

파일 크기가 0이면 변환 중 오류가 발생한 거예요.

**해결 방법:**

먼저 변환 명령어를 직접 실행해서 오류 메시지를 확인해요.

```bash
npx md-to-pdf output/resume-kakao.md && mv output/resume-kakao.pdf output/pdf/
```

출력되는 메시지를 보고 원인을 파악하세요. Chromium 관련 오류라면 앞의 항목을 참고하고, 파일 경로 오류라면 경로가 올바른지 확인해요.

`output/pdf/` 폴더 자체가 없어서 저장에 실패하는 경우도 있어요. 폴더를 직접 만들어 주세요.

```bash
# macOS/Linux
mkdir -p output/pdf

# Windows
mkdir output\pdf
```

> **TIP:** PDF 변환이 계속 실패한다면 브라우저를 활용하는 게 가장 빠른 해결책이에요. VS Code에서 마크다운 파일을 열고, `Cmd+Shift+V` (macOS) / `Ctrl+Shift+V` (Windows)로 미리보기를 연 다음, 미리보기 창에서 마우스 우클릭 → **"인쇄"** → **"PDF로 저장"**을 선택하면 깔끔한 PDF가 만들어져요.

## 그래도 해결이 안 된다면?

여기서 다루지 않은 오류가 나오거나, 위의 방법을 모두 시도해도 해결이 안 된다면 아래 방법을 활용해 보세요.

**Claude Code에게 직접 물어보세요.**

Claude Code 자체가 훌륭한 오류 해결사예요. claude를 실행하고 오류 메시지를 그대로 붙여넣으면서 물어보면 대부분 친절하게 안내해 줘요.

```
터미널에 이런 오류가 나왔어요:
[오류 메시지 붙여넣기]

macOS를 사용하고 있고, Node.js 버전은 v20이에요.
어떻게 해결하면 될까요?
```

운영체제와 Node.js 버전 같은 환경 정보를 함께 알려주면 더 정확한 도움을 받을 수 있어요.

**공식 문서를 확인하세요.**

Claude Code 공식 문서: [docs.anthropic.com/en/docs/claude-code](https://docs.anthropic.com/en/docs/claude-code)

릴리즈 노트나 알려진 이슈 목록에서 현재 겪는 문제와 같은 내용을 찾을 수도 있어요.

**검색 엔진을 활용하세요.**

오류 메시지의 핵심 부분을 그대로 복사해서 구글이나 네이버에 붙여넣어 보세요. 같은 문제를 먼저 겪고 해결한 분들의 경험을 Stack Overflow, GitHub Issues, 개발 블로그 등에서 찾을 수 있어요.

> **WARNING:** 검색하다 보면 "API 키를 발급받아서 `ANTHROPIC_API_KEY` 환경 변수에 등록하라"는 안내를 종종 만날 수 있어요. 이 책에서 만드는 파이프라인은 API 키가 아닌 **OAuth 로그인 방식**(`claude auth login`)을 사용해요. API 키 설정은 필요 없으니, 그런 가이드는 이 파이프라인에 적용하지 않도록 주의하세요.

## 자주 묻는 질문 (FAQ)

### Claude Pro와 Max, 어떤 플랜을 골라야 하나요?

이 책의 모든 실습은 **Pro 플랜($20/월)**으로 충분해요. Max 플랜($100/월)은 더 많은 사용량과 빠른 응답을 제공하지만, 자소서 파이프라인을 만들고 사용하는 데는 Pro면 충분해요.

### 사용량 제한에 걸렸어요. 어떻게 하나요?

Claude Pro 플랜은 시간당/일당 사용 가능한 양에 한도가 있어요. 한도에 도달하면 이런 메시지가 나타날 수 있어요:

> "You've reached your usage limit. Please wait before sending more messages."

걱정하지 마세요! 잠시 기다리면(보통 몇 시간 이내) 다시 사용할 수 있어요. 파이프라인 실습을 한 번에 몰아서 하기보다는, 챕터별로 나눠서 진행하면 제한에 걸릴 확률이 낮아져요.

### 비전공자인데 따라할 수 있나요?

네, 충분히 가능해요! 이 책은 터미널을 한 번도 써본 적 없는 분을 기준으로 썼어요. 프로그래밍 지식이 없어도 괜찮아요. 0장의 설치 과정부터 차근차근 따라오면 돼요.

### AI가 쓴 자소서를 그대로 제출해도 되나요?

AI가 생성한 초안을 **그대로** 제출하는 건 권장하지 않아요. 이 파이프라인의 목적은 "AI가 대신 써주는 것"이 아니라, **"내 경험을 효과적으로 정리하는 도구"**로 활용하는 거예요.

추천하는 활용법:
- AI 초안을 **뼈대**로 활용하고, 자신의 말투와 표현으로 다듬기
- 환각 검출(`/review-resume`)로 사실과 다른 내용 걸러내기
- 최종 제출 전에 반드시 본인이 한 번 더 읽고 수정하기

### 개발 직군이 아닌데도 쓸 수 있나요?

네! 예시는 백엔드 개발자 중심이지만, 마케팅, 기획, 디자인 등 **어떤 직군이든** 동일한 파이프라인을 사용할 수 있어요. 달라지는 건 두 가지뿐이에요:

1. `data/my-profile.md`에 자신의 직군에 맞는 경험을 작성
2. JD를 자신이 지원하는 직군의 채용공고로 교체

나머지 스킬들은 직군에 관계없이 동일하게 작동해요.

### JD가 영어인데 어떻게 하나요?

외국계 기업의 영어 JD도 그대로 `data/jd/` 폴더에 저장하면 돼요. Claude는 영어 JD를 분석하고, 한국어 자소서를 생성할 수 있어요. `/collect-jd` 실행 시 "한국어로 분석해줘"라고 추가하면 분석 결과도 한국어로 받을 수 있어요.

영문 커버레터가 필요하다면 부록 B의 영문 커버레터 템플릿을 참고하세요.

### Claude Code를 업데이트하려면 어떻게 하나요?

터미널에서 아래 명령어를 실행하면 최신 버전으로 업데이트돼요:

```bash
npm update -g @anthropic-ai/claude-code
```

업데이트 후 `claude --version`으로 버전이 바뀌었는지 확인해보세요.

### 대화가 너무 길어졌을 때는 어떻게 하나요?

Claude Code에서 유용한 명령어 몇 가지를 알아두면 편해요:

| 명령어 | 설명 |
|--------|------|
| `/clear` | 현재 대화를 초기화해요 (새로운 대화 시작) |
| `/compact` | 긴 대화를 요약해서 압축해요 |
| `Shift + Tab` | 입력 중 줄바꿈 (여러 줄 입력할 때) |
| `Esc` | 현재 입력을 취소해요 |
| `/exit` | Claude Code를 종료해요 |
