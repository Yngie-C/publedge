# 맞춤형 자소서 생성: 두 번째 스킬

지난 챕터까지 우리는 채용공고를 분석해서 핵심 키워드를 뽑아내는 파이프라인을 만들었어요. 이번 챕터에서는 그 분석 결과를 실제 자소서로 연결하는 두 번째 스킬을 만들어볼 거예요.

---

## Ch 1 vs Ch 4: 무엇이 달라지나요?

1챕터에서 처음 만들었던 자소서 기억하시나요? 당시에는 클로드에게 "자소서를 써줘"라고 하면 클로드가 가진 일반적인 지식으로 자소서를 작성해줬어요. 나쁘지 않은 결과물이지만, 솔직히 말하면 **어디서나 볼 수 있는 평범한 자소서**였죠.

이번 챕터에서 만드는 건 완전히 달라요.

| 구분 | Ch 1 자소서 | Ch 4 자소서 |
|------|------------|------------|
| 입력 | 없음 (클로드 지식만 사용) | JD 분석 JSON + 내 경력기술서 |
| 키워드 | 일반적인 취업 키워드 | 해당 회사·직무 특화 키워드 |
| 내 경험 반영 | 불가능 | 내 프로젝트·수치 그대로 반영 |
| 결과물 | 누가 써도 비슷한 자소서 | 나만의 맞춤형 자소서 |

옷에 비유하면 이렇게 돼요. Ch 1은 "오늘 뭐 입지?" 하면서 아무 옷이나 입는 것이에요. Ch 4는 "이 회사 면접은 비즈니스 캐주얼이라는데, 내 옷장에서 뭐가 맞지?" 하고 드레스코드를 확인하고 그에 맞는 옷을 고르는 것이죠.

채용 담당자는 하루에 수십, 수백 개의 자소서를 읽어요. **JD에 있는 키워드가 자연스럽게 녹아든 자소서**는 그 중에서 금방 눈에 띄어요. 그게 바로 우리가 목표로 하는 결과물이에요.

---

## 내 경험 정리하기 — 경력기술서 만들기 (중요!)

> **💡 TIP:** 2장에서 CLAUDE.md에도 간단한 정보를 적었지만, 여기서 만드는 경력기술서는 훨씬 상세한 버전이에요. CLAUDE.md는 Claude에게 "나는 이런 사람이야"라고 알려주는 간단한 소개이고, `data/my-profile.md`는 스킬들이 자소서를 쓸 때 참조하는 **공식 원본 재료**예요.

본격적으로 스킬을 만들기 전에, 반드시 먼저 해야 할 작업이 있어요. 바로 **내 경력기술서 파일**을 만드는 것이에요.

이 파일의 경로는 `data/my-profile.md`예요.

> **WARNING:** `data/my-profile.md` 파일이 없으면 `/generate-resume` 스킬이 제대로 작동하지 않아요. 스킬이 자소서를 쓸 때 이 파일을 "원본 재료"로 사용하거든요. 반드시 먼저 만들어주세요!

이 파일은 **한 번만 잘 작성하면 앞으로 모든 자소서에 재사용**돼요. 시간을 좀 더 들여서 구체적으로 작성할수록 나중에 생성되는 자소서의 질이 높아져요. 아래 템플릿을 기반으로 작성해보세요.

### 경력기술서 템플릿

```markdown
# 내 프로필

## 기본 정보
- 이름: (본명)
- 전공: (대학교, 졸업 예정일)

## 프로젝트 경험
- [프로젝트명]: [기간], [역할], [성과 수치]
  - 예: "쇼핑몰 웹앱 개발: 2025.03-2025.06, 백엔드 리드, 일일 주문 처리 300% 향상"

## 기술 스택
- 언어: Python, JavaScript, ...
- 프레임워크: React, Django, ...
- 도구: Git, Docker, ...

## 자격증/수상
- [자격증명] (취득일)
- [수상명] (대회명, 날짜)

## 강점/성격
- [키워드]: [구체적 에피소드 1-2문장]
  - 예: "끈기: 졸업 프로젝트에서 3번의 기술 변경에도 마감일을 지켰습니다"
```

> **TIP:** 성과 수치가 없다면 억지로 만들 필요는 없어요. 다만 "기능을 구현했다"보다는 "팀장으로서 주간 스프린트를 주도해 3주 만에 MVP를 배포했다"처럼 **행동과 맥락을 구체적으로** 쓰면 클로드가 훨씬 좋은 자소서를 만들어줘요.

### 실제 예시 — 김지원의 프로필

막막하다면 이 예시를 참고해서 여러분의 경험으로 채워보세요. 아래는 가상 인물 김지원 씨의 프로필이에요.

```markdown
# 내 프로필

## 기본 정보
- 이름: 김지원
- 전공: 컴퓨터공학과, 한국대학교 (2026년 2월 졸업 예정)

## 프로젝트 경험
- 중고 도서 거래 플랫폼 "북마켓" 개발: 2025.03-2025.08, 백엔드 개발 리드 (4인 팀)
  - Django REST Framework로 API 25개 설계·구현
  - Redis 캐싱 도입으로 상품 목록 조회 응답속도 68% 개선 (1.2s → 0.4s)
  - GitHub Actions로 CI/CD 파이프라인 구축, 배포 시간 40분 → 8분 단축
  - 월간 활성 사용자 350명, 누적 거래 1,200건 달성

- 실시간 위치 공유 앱 "여기있어" 개발: 2024.09-2024.12, 풀스택 개발 (2인 팀)
  - React Native + FastAPI 스택으로 iOS/Android 동시 출시
  - WebSocket 기반 실시간 위치 동기화 구현 (지연 300ms 이하)
  - 앱스토어 출시 후 4주 만에 500 다운로드 달성

- 교내 강의 평가 시스템 리팩토링: 2024.03-2024.06, 기여자 (오픈소스 기여)
  - 레거시 jQuery 코드를 React로 마이그레이션
  - 로딩 시간 2.1s → 0.6s 개선, PR 머지 완료

## 기술 스택
- 언어: Python, JavaScript/TypeScript, Java, SQL
- 백엔드: Django, FastAPI, Spring Boot (학습 중)
- 프론트엔드: React, React Native
- 데이터베이스: PostgreSQL, MySQL, Redis
- 인프라/도구: Docker, GitHub Actions, AWS EC2/S3, Git

## 자격증/수상
- 정보처리기사 (2025.06 취득)
- SW마에스트로 14기 수료 (2024.11)
- 교내 해커톤 "코딩런" 최우수상 (한국대학교, 2024.05)
- TOEIC 845점 (2025.01)

## 강점/성격
- 문제 해결 집중력: 북마켓 프로젝트에서 DB 쿼리 병목을 혼자 3일간 분석해
  N+1 문제를 발견하고 select_related 적용으로 쿼리 수를 89개에서 4개로 줄였어요.
- 팀 커뮤니케이션: 주 2회 팀 회고를 직접 운영하며 이슈를 조기에 공유하는 문화를 만들었고,
  팀원들이 "같이 일하고 싶은 동료"로 꼽아줬어요.
- 빠른 학습: WebSocket을 처음 써보면서도 2주 만에 실시간 기능을 완성했고,
  이후 팀 내부 세션에서 해당 기술을 발표했어요.
```

> **NOTE:** 자격증이 없거나 수상 경력이 없어도 괜찮아요. 없는 항목은 비워두거나 해당 섹션을 삭제하면 돼요. 클로드는 있는 재료만으로 최선의 자소서를 작성해줄 거예요.

## /generate-resume 스킬 만들기 — build-then-reveal

이제 본격적으로 스킬 파일을 만들 차례예요. 먼저 스킬이 어떻게 작동할지 핵심 원리를 이해하고, 그다음 전체 파일을 작성할게요.

### 핵심 먼저 이해하기

이 스킬에는 세 가지 핵심 지시사항이 들어가요.

**1. "당신은 취업 컨설턴트입니다" — 역할 지정**

클로드에게 역할을 명확하게 줄수록 결과물이 달라져요. "자소서를 써줘"보다 "당신은 10년 경력의 취업 컨설턴트입니다. 지원자의 강점을 극대화하는 자소서를 작성해주세요."가 훨씬 더 좋은 결과를 내요.

**2. "$ARGUMENTS에서 JSON 경로를 받고, data/my-profile.md도 함께 읽어주세요" — 두 파일 동시 참조**

`$ARGUMENTS`는 스킬 실행 시 사용자가 입력한 값이에요. `/generate-resume data/analysis/techcorp-backend.json`이라고 실행하면 `$ARGUMENTS`가 `data/analysis/techcorp-backend.json`이 돼요. 스킬은 이 파일과 `data/my-profile.md`를 동시에 읽어서 두 정보를 결합해요.

**3. "JD의 핵심 키워드를 자연스럽게 포함할 것" — JD 맞춤화의 핵심**

이게 바로 Ch 1과 Ch 4의 가장 큰 차이점이에요. 단순히 키워드를 끼워넣는 게 아니라, **내 경험을 설명하는 문장 안에 자연스럽게 녹아들게** 쓰는 것이 핵심이에요.

### 전체 스킬 파일 작성

`.claude/commands/generate-resume.md` 파일을 만들어요.

~~~markdown
# Generate Resume

당신은 10년 경력의 취업 컨설턴트입니다. 지원자의 경험과 강점을 최대한 부각하는 맞춤형 자소서를 작성하는 전문가예요.

## 입력 처리

1. `$ARGUMENTS`에서 JD 분석 JSON 파일 경로를 읽어주세요.
2. `data/my-profile.md` 파일도 함께 읽어주세요.
3. 두 파일을 모두 읽은 후 자소서 작성을 시작하세요.

## 작성 원칙

1. **JD 키워드 자연스러운 포함**: JSON의 `keywords` 배열에 있는 단어들을 문장 안에 자연스럽게 녹여 쓰세요. 키워드를 단순히 나열하지 말고, 내 경험을 설명하는 흐름 안에 배치하세요.
2. **구체적 수치와 성과 활용**: `data/my-profile.md`의 프로젝트 경험에서 수치와 성과를 그대로 인용하세요. 수치가 없는 경험은 행동과 맥락을 구체적으로 서술하세요.
3. **회사 특성 반영**: JSON의 `culture`를 참고해 톤을 맞추세요. 스타트업이면 실행력과 주도성, 대기업이면 협업과 안정성을 강조하세요.
4. **개인적인 어조 유지**: "~했습니다" 체로 작성하되, 딱딱하지 않고 진정성 있게 쓰세요. 클리셰(예: "항상 최선을 다하는")는 사용하지 마세요.
5. **항목별 분량**: 각 항목은 300-400자를 기준으로 하되, 내용에 따라 ±50자는 허용해요.
6. **두괄식 구성**: 각 항목의 첫 문장에 핵심 메시지를 담아주세요.

## 자소서 구조

### 1. 지원동기 (300-400자)
- JSON의 `company`와 `position`을 언급하세요.
- 해당 회사·직무를 선택한 구체적 이유를 쓰세요 (단순 "성장 가능성" 같은 표현 금지).
- `culture`나 `tech_stack`에서 끌어올 수 있는 요소를 활용하세요.

### 2. 직무역량 (350-450자)
- 지원 직무와 가장 관련 있는 프로젝트 경험 1-2개를 중심으로 쓰세요.
- `keywords`에서 직무 관련 키워드를 자연스럽게 포함하세요.
- 수치 성과가 있으면 반드시 포함하세요.

### 3. 성장과정 (250-350자)
- 개발자로서의 성장 궤적을 보여주는 에피소드 하나를 중심으로 쓰세요.
- `my-profile.md`의 강점/성격 섹션에서 적합한 에피소드를 골라주세요.

### 4. 입사 후 포부 (250-350자)
- 단기(1-2년)와 장기(3-5년) 목표를 구분해서 쓰세요.
- JSON의 `required`와 `preferred`에서 성장하고 싶은 방향을 연결하세요.
- 회사에서 내가 기여할 수 있는 구체적 방향을 제시하세요.

## 출력

아래 형식으로 `output/resume-{회사명}-{직무명}.md` 파일을 생성해주세요.
파일명은 소문자 영어로, 공백은 하이픈(-)으로 연결하세요.
(예: `output/resume-techcorp-backend.md`)

```markdown
# [회사명] [직무명] 자소서

> 작성일: [오늘 날짜]
> 기반 JD 분석: [JSON 파일 경로]

---

## 1. 지원동기

[내용]

---

## 2. 직무역량

[내용]

---

## 3. 성장과정

[내용]

---

## 4. 입사 후 포부

[내용]
```

파일 생성 후, 각 항목의 실제 글자 수와 포함된 주요 키워드 목록을 간단히 요약해서 알려주세요.
~~~

> **NOTE:** 스킬 파일 안에 마크다운 코드블록(``` ` ` ` ```)을 포함해야 해서, 바깥쪽은 `~~~`를 사용했어요. 실제 파일에 저장할 때는 가장 바깥의 `~~~` 없이 안쪽 내용만 저장하면 돼요.

## 스킬 실행 & 결과 확인

스킬 파일을 저장했다면 이제 실행해볼 거예요. 현재 프로젝트 폴더에서 클로드 코드를 열고 아래 명령어를 입력해요.

```bash
/generate-resume data/analysis/techcorp-backend.json
```

클로드가 실행하면서 다음 순서로 작업해요.

1. `data/analysis/techcorp-backend.json` 파일을 읽어요.
2. `data/my-profile.md` 파일을 읽어요.
3. 두 파일의 내용을 결합해서 자소서를 작성해요.
4. `output/resume-techcorp-backend.md` 파일을 생성해요.
5. 각 항목의 글자 수와 포함된 키워드 목록을 요약해줘요.

실행이 끝나면 아래처럼 요약 결과가 터미널에 출력될 거예요.

```
output/resume-techcorp-backend.md 파일을 생성했어요.

항목별 글자 수:
- 지원동기: 342자
- 직무역량: 418자
- 성장과정: 298자
- 입사 후 포부: 315자

포함된 주요 키워드:
- 백엔드 개발, RESTful API, 성능 최적화, 코드 리뷰, 협업, CI/CD, 확장성
```

### 결과 파일 직접 확인하기

이제 `output/resume-techcorp-backend.md` 파일을 열어서 내용을 살펴봐요. 확인할 포인트는 세 가지예요.

**1. JD 키워드가 자연스럽게 녹아들었는지**

"저는 RESTful API를 설계하는 역량을 보유하고 있습니다"처럼 키워드가 억지로 끼워진 느낌이 든다면 프롬프트를 수정해야 해요. 반면 "북마켓 프로젝트에서 25개의 RESTful API를 설계하며 응답속도를 68% 개선한 경험은..." 처럼 자연스럽게 녹아있다면 잘 된 거예요.

**2. 내 실제 수치가 포함되었는지**

경력기술서에 적어둔 수치(예: "응답속도 68% 개선", "월간 활성 사용자 350명")가 자소서에 그대로 반영되어 있는지 확인해요. 수치 없이 일반적인 표현만 있다면, `my-profile.md`에서 수치를 더 명확하게 표기하거나 스킬의 "구체적 수치와 성과 활용" 원칙을 강화하면 돼요.

**3. 분량이 적절한지**

각 항목이 너무 짧거나(200자 미만) 너무 길다면(500자 초과) 스킬의 분량 지침을 조정하면 돼요.

> **TIP:** 처음 실행한 결과물이 완벽하지 않아도 걱정하지 마세요. 다음 섹션에서 스킬 파일을 수정해서 원하는 방향으로 다듬는 방법을 알아볼 거예요.

## 1장 vs 4장: 자소서가 이렇게 달라져요

1장에서 만든 자소서와 지금 만든 자소서, 어떤 차이가 있는지 비교해볼까요?

### Before: 1장에서 만든 자소서 (일반 버전)

> **지원동기**
>
> 저는 대학교 4년간 컴퓨터공학을 전공하며 백엔드 시스템에 깊은 관심을 가져왔습니다. 특히 대용량 트래픽을 처리하는 분산 시스템 설계에 매력을 느꼈고, 테크코프가 국내 최대 규모의 실시간 데이터 처리 인프라를 운영한다는 점에서 입사를 희망하게 되었습니다.

### After: 4장에서 만든 자소서 (맞춤형 버전)

> **지원동기**
>
> 테크코프의 대규모 트래픽 처리 기술에 매력을 느꼈습니다. 북마켓 프로젝트에서 Django REST Framework로 API 25개를 설계하고 Redis 캐싱을 도입해 응답속도를 68% 개선한 경험이 있습니다. 특히 귀사가 채용공고에서 강조한 "확장 가능한 백엔드 아키텍처"는 제가 CI/CD 파이프라인을 직접 구축하며 가장 몰입했던 주제예요. 그 경험을 실제 서비스 규모에서 펼쳐보고 싶어 지원하게 됐습니다.

### 무엇이 달라졌나요?

| 구분 | 1장 (일반) | 4장 (맞춤형) |
|------|-----------|-------------|
| 회사 정보 | "국내 최대 규모의 인프라" (막연한 표현) | "대규모 트래픽 처리 기술" (JD에서 추출한 구체적 정보) |
| 내 경험 | 전공과 관심사만 언급 | "API 25개 설계, 응답속도 68% 개선" (구체적 수치) |
| JD 키워드 | 없음 | "확장 가능한 백엔드 아키텍처" (JD 원문 반영) |
| 연결고리 | 회사와 내 경험이 따로 놀아요 | 회사가 원하는 것 + 내가 한 것이 연결돼요 |

> **TIP:** 이 차이를 만드는 건 JD 분석 JSON과 경력기술서예요. 같은 AI라도 **어떤 재료를 주느냐**에 따라 결과가 완전히 달라져요.

## 프롬프트 튜닝하기

이게 클로드 코드를 쓰는 가장 큰 장점 중 하나예요. 결과물이 마음에 안 들면 스킬 파일을 조금 수정하고 다시 실행하면 돼요. Word 문서나 ChatGPT처럼 매번 새로 입력할 필요가 없어요.

### 튜닝 예시 1: 문장이 너무 길어요

자소서를 읽었을 때 문장이 지나치게 길고 복잡하다면, 스킬의 "작성 원칙"에 아래 한 줄을 추가해요.

```markdown
7. **간결한 문장**: 한 문장은 50자를 넘지 않게 쓰세요. 내용이 길어지면 문장을 나누세요.
```

수정 후 다시 `/generate-resume data/analysis/techcorp-backend.json`을 실행하면 훨씬 간결한 버전이 만들어져요.

### 튜닝 예시 2: 너무 딱딱한 느낌이에요

"~하였습니다", "~하였으며" 같은 경직된 표현이 많다면 이렇게 수정해요.

```markdown
# 변경 전
4. **개인적인 어조 유지**: "~했습니다" 체로 작성하되, 딱딱하지 않고 진정성 있게 쓰세요.

# 변경 후
4. **개인적인 어조 유지**: "~했습니다" 체로 작성하되, 딱딱하지 않고 진정성 있게 쓰세요.
   "~하였습니다", "~함으로써" 같은 경직된 표현은 피하고 자연스러운 구어체에 가깝게 쓰세요.
```

### 튜닝 예시 3: 지원동기가 너무 뻔해요

지원동기 섹션이 "귀사의 성장 가능성에 매력을 느껴..." 같은 클리셰로 채워진다면, 해당 섹션 지침을 더 구체화해요.

```markdown
### 1. 지원동기 (300-400자)
- JSON의 `company`와 `position`을 언급하세요.
- 해당 회사·직무를 선택한 구체적 이유를 쓰세요.
- "성장 가능성", "비전", "발전" 같은 모호한 단어는 사용하지 마세요.
- 대신 회사의 구체적인 기술 스택이나 서비스 특성을 언급하세요.
  예: "테크코프의 마이크로서비스 아키텍처에서 제가 북마켓 프로젝트에서 쌓은 API 설계 경험을 직접 적용할 수 있다고 생각했어요."
```

> **TIP:** 스킬 파일을 조금씩 수정하면서 최적의 프롬프트를 찾는 과정 자체가 핵심 스킬이에요. 처음부터 완벽한 프롬프트를 만들려고 하지 말고, 결과물을 보면서 어느 부분이 아쉬운지 파악하고 그 부분만 수정하는 방식으로 접근해보세요. 보통 3-4번 수정하면 꽤 만족스러운 결과가 나와요.

### 프롬프트 변경 이력 관리하기

여러 번 수정하다 보면 "이전 버전이 더 나았는데..."라는 상황이 생겨요. 이럴 때를 대비해서 수정할 때마다 간단히 커밋해두면 편해요.

```bash
git add .claude/commands/generate-resume.md
git commit -m "자소서 스킬: 문장 길이 제한 추가"
```

이렇게 해두면 언제든지 이전 버전으로 돌아갈 수 있어요.

## 지금까지 만든 것 정리

이번 챕터를 마치고 나면 프로젝트 구조가 이렇게 돼요.

```
my-resume-pipeline/
  CLAUDE.md
  resume-v1.md
  data/
    jd/
      techcorp-backend.md
    analysis/
      techcorp-backend.json
    my-profile.md                  ← 이번 챕터에서 만든 파일
  output/
    resume-techcorp-backend.md     ← 자동 생성된 자소서
  .claude/
    commands/
      hello.md
      collect-jd.md
      generate-resume.md           ← 이번 챕터에서 만든 스킬
```

Ch 3에서 만든 `techcorp-backend.json`과 이번에 만든 `my-profile.md`가 합쳐져서 드디어 실제 자소서(`resume-techcorp-backend.md`)가 완성됐어요.

파이프라인의 전체 흐름을 다시 한번 정리하면 이렇게 돼요.

```
채용공고 텍스트 (data/jd/techcorp-backend.md)
    ↓ /collect-jd
분석 결과 JSON (data/analysis/techcorp-backend.json)
    ↓ /generate-resume (+ data/my-profile.md 함께 참조)
맞춤형 자소서 (output/resume-techcorp-backend.md)
```

다음 챕터에서는 AI가 만든 자소서를 꼼꼼하게 검토하는 방법을 배울 거예요. AI가 사실과 다르게 지어낸 내용은 없는지 환각 검출, 100점 만점 채점, 그리고 반복 개선 워크플로우까지 익혀볼 거예요. 기대해주세요!
