# 전자책 콘텐츠 종합 리뷰

이 전자책의 목적은 두 가지입니다:
1. **클로드 코드(Claude Code) 학습** - CLI 도구로서의 Claude Code 사용법
2. **자소서 파이프라인 구축** - 채용공고 분석 → 자소서 생성 → 검토 → PDF 출력

`content/` 디렉토리의 모든 `.md` 파일과 `content/skill-specs/` 디렉토리의 모든 파일을 대상으로 아래 4가지 관점의 리뷰를 병렬로 수행하세요.

---

## 1. 콘텐츠 적합성 검토

위 두 가지 목적에 비추어:
- 목적과 관련 없는 내용이 포함되어 있다면 **제거 또는 수정을 제안**하세요.
- 목적 달성에 필요하지만 빠져 있는 내용이 있다면 **보완을 제안**하세요.
- 각 챕터 간 참조(예: "3장에서 만든 스킬을")가 실제 내용과 맞는지 확인하세요.
- 챕터 말미의 "다음 챕터 미리보기" 문구가 실제 다음 챕터 내용과 일치하는지 확인하세요.

## 2. 기술적 정확성 검토

**반드시 최신 상태와 대조하여** 다음을 확인하세요:

- **Claude Code CLI**: 명령어(`claude auth login` 등), 플래그, 옵션이 현재 버전에서 유효한지. Deprecated되거나 이름이 바뀐 것은 없는지.
- **npm 패키지**: `@anthropic-ai/claude-code`, `md-to-pdf` 등 패키지명과 사용법이 정확한지. 존재하지 않는 플래그/옵션을 사용하고 있지는 않은지.
- **파일 구조**: `.claude/commands/` 경로, `CLAUDE.md` 위치, skill 파일의 형식이 현재 Claude Code 스펙과 맞는지.
- **코드 예시**: 제시된 Shell 명령어, Node.js 코드가 실제로 동작하는지.
- **개념 설명**: Context, Skill(슬래시 커맨드), Hook, MCP 등의 동작 방식이 정확히 설명되었는지.
- **skill-specs/ 와 본문 스킬 파일 내용 일치 여부**: 본문에서 독자가 만드는 스킬 파일과 skill-specs의 정본이 동일한지.
- **JSON 필드명 일관성**: `/collect-jd`가 생성하는 JSON 구조와 `/generate-resume`, `/review-resume`가 참조하는 필드명이 일치하는지.

각 문제에 대해 **파일명:라인번호**, 문제 내용, 올바른 내용을 구체적으로 제시하세요.

## 3. 오탈자 및 문맥 검토

- **맞춤법/띄어쓰기 오류**
- **문맥에 어긋나는 표현** (앞뒤 논리 불일치, 비약)
- **용어 일관성**: "스킬"로 통일되어 있는지 (ch02에서 정의한 용어 규칙 준수 여부). 정의/소개 문맥 외에서 "슬래시 커맨드"가 사용되고 있지 않은지.
- **문체 일관성**: 전자책 독자 대상(취준생, AI 도구 초보자)에 적합한 구어체("~이에요", "~해요")가 유지되는지.
- **숫자/순서**: 챕터 번호, 스킬 이름 등 교차 참조가 정확한지.

## 4. 보완 및 보조 자료 제안

- **스크린샷/이미지가 필요한 곳**: 텍스트만으로 이해하기 어려운 부분 (터미널 화면, 설치 과정, 결과물 등)
- **다이어그램이 필요한 곳**: 파이프라인 흐름도, 데이터 흐름, 개념 설명 등
- **추가 콘텐츠**: 독자가 궁금해할 FAQ, 실전 팁, Before/After 예시 등
- **학습 보조 장치**: 요약 박스, 체크리스트, 연습 문제 등

---

## 출력 형식

각 관점별로 발견 사항을 **심각도(CRITICAL / HIGH / MEDIUM / LOW)** 순으로 정리하고, 마지막에 **수정 우선순위 표**를 제시하세요.

수정이 필요한 항목은 직접 수정하고, 직접 수정이 어려운 항목(스크린샷 촬영, 다이어그램 제작 등)은 상세한 가이드를 제공하세요.
