# Agent Knowledge

이 디렉터리는 Publedge 프로젝트에서 Hermes, Codex, Claude Code, OMX, OMC, Ollama-Cloud 기반 오픈소스 모델 등 다양한 코딩 에이전트가 함께 참조할 **장기 프로젝트 지식의 canonical 저장소**입니다.

## 목적

- 모델 제공자나 하네스가 바뀌어도 유지되어야 하는 프로젝트 지식을 한곳에 모읍니다.
- `.claude/`, `.omc/`, `.omx/`에 흩어진 과거 지식 중 계속 유효한 내용을 선별해 요약합니다.
- 새로 발견한 컨벤션, 아키텍처 결정, 워크플로, 환경 설정 지식을 Git으로 추적 가능한 문서로 남깁니다.

## 지식 저장 우선순위

1. `AGENTS.md`
   - 코딩 에이전트의 최상위 운영 계약서입니다.
   - 작업 규칙, 보안 규칙, 검증 기준, Git 규칙은 여기를 우선합니다.
2. `docs/agent-knowledge/`
   - 장기 보존할 프로젝트 지식의 canonical 저장소입니다.
   - 새 지식은 기본적으로 이 디렉터리에 추가하거나 기존 문서를 갱신합니다.
3. `.claude/`, `.omc/`
   - Claude Code / oh-my-claudecode에서 넘어온 legacy 참고 자료입니다.
   - 읽기 전용 historical context로 취급하고, 새 지식은 여기에 추가하지 않습니다.
4. `.omx/`
   - OMX 런타임 상태, 로컬 노트, 계획, 로그, 캐시가 들어갈 수 있는 도구별 작업 공간입니다.
   - 장기 지식의 canonical 저장소로 취급하지 않습니다.

## 새 지식을 남기는 기준

다음에 해당하면 이 디렉터리에 문서로 남기세요.

- 반복해서 참조할 코딩 컨벤션 또는 프로젝트 운영 규칙
- 아키텍처 결정과 그 이유
- 도구/Hermes/Ollama-Cloud/OMX 설정 방법
- 배포, 테스트, Supabase, 콘텐츠 제작 등 프로젝트 특화 워크플로
- `.claude/`, `.omc/`, `.omx/`에서 발견한 내용 중 앞으로도 유효한 요약

다음은 이 디렉터리에 남기지 않아도 됩니다.

- 일회성 세션 로그
- 임시 디버깅 흔적
- 자동 생성 캐시
- secret 또는 실제 환경변수 값
- 오래되어 더 이상 유효하지 않은 계획 전체 사본

## 권장 파일 구성

필요할 때 아래 파일을 추가하세요.

- `conventions.md` — 코딩/문서/콘텐츠 컨벤션
- `architecture.md` — 주요 구조와 설계 결정
- `workflows.md` — 반복 작업 절차
- `environment.md` — 로컬/하네스/모델 제공자 설정
- `decisions.md` — 장기 의사결정 기록

## 현재 하네스 방향

이 프로젝트는 Hermes를 코딩 에이전트 하네스로 사용하면서, 필요에 따라 GPT, Claude, Ollama-Cloud 기반 오픈소스 모델 등 여러 모델 제공자를 붙일 수 있습니다.

모델 제공자는 프로젝트 지식의 source of truth가 아닙니다. 에이전트는 항상 `AGENTS.md`를 먼저 읽고, 장기 지식은 이 디렉터리에서 확인합니다.
