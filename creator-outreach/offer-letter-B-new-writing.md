# 오퍼 B — 업무 노하우 신규 집필 영입 레터

## 용도

**대상**: 위키독스에 기술서를 완주한 경험이 있고, 실무 배경이 명확한 저자 (현직 개발자·PM·디자이너 등)
**제안**: 기존 책과 무관한 **업무 노하우 책을 새로 집필**. publedge 전용 오리지널 콘텐츠.
**핵심 메시지**: "당신이 실무에서 10년간 쌓은 것을 책으로. 제작 과정은 저희가 함께."

## 톤 가이드

**✅ 하기**
- 저자 기존 책에서 구체 챕터·통찰 인용
- "저자님이 고르시면 된다", "제안은 방향일 뿐" — 주제 선택권 존중
- 제작 프로세스 구체화 (에디터 붙음, 킥오프 방식)
- 분량·기간을 현실적으로 제시 (과도하게 적게 잡으면 불성실, 과도하게 길면 부담)

**❌ 피하기**
- "베스트셀러 만들어드리겠다" 류 허언
- 주제를 못 박아 제안 ("이거 써주세요" → 창작 자율성 침해 인상)
- 기존 책 폄하 ("기존 책보다 더 나은 것을 써주세요")
- 저자 이력을 잘못 추정 (LinkedIn 없으면 차라리 추정 없이 감)

## 오퍼 B 특유의 설득 포인트

1. **"책 한 권을 써본 적 있으니 새 책 쓰는 게 어렵지 않다"** 프레이밍 → 허들 낮춤
2. **"강의 플랫폼보다 시간 투입 적음"** → 강의 촬영 vs 책 집필 비교
3. **"이미 있는 노하우를 글로 옮기기만 하면 된다"** → 새로 배워야 하는 게 없음 강조
4. **"당신의 커리어 자산"** → 판매 수익 외에 "책 저자 타이틀"이 주는 커리어 가치 언급

## 오퍼 B의 가장 큰 실패 지점

**실패 유형**: 계약까지는 성사됐지만 원고가 3개월 안 나옴 → 영원히 지연

**예방책**:
1. 킥오프 미팅에서 **주당 고정 집필 시간 약속** 받기 ("매주 화요일 밤 2시간" 같은 구체 슬롯)
2. 2주마다 짧은 진도 체크인 (30분 콜 or 슬랙)
3. 1차 초고 데드라인 명시 (계약서에 soft deadline)
4. 첫 2주 안에 최소 1장 분량 써보기 과제 — 안 나오면 조기 캔슬 가능성 판단
