# 위키독스 저자 영입 플레이북

publedge 초기 베타 창작자 영입을 위한 위키독스(wikidocs.net) 저자 프로파일링·컨택 가이드.

## 0. 전략 요약

**목표**: 초기 카탈로그 30~40권 확보 (베타 창작자 20~30명 영입)

**핵심 포지셔닝**: "업무 노하우 × 인터랙티브 워크북"

**티어 구성**

| 티어 | 비율 | 인원 | 역할 |
|---|---|---|---|
| **anchor** (상위 저자) | 5~10% | 1~3명 | 신뢰성 상징, 초기 바이럴 |
| **mid** (중간대) | 40% | 10~15명 | 카탈로그 주력 |
| **underdog** (발굴형) | 50% | 15~20명 | 숨은 보석, 로열티 자산 |

**왜 underdog 비중이 높은가**: 영입 전환율이 훨씬 높고, 플랫폼 혜택의 상대적 체감가치가 크며, 콘텐츠 퀄리티는 좋은데 위키독스 상위 노출 운이 없었던 숨은 보석이 많기 때문.

## 1. 롱리스트 추출 방법

### 탐색 동선 (wikidocs.net)

- **anchor 후보**: 메인 → "주간 베스트 TOP 100" 중 추천수 1,000+ & 전자책 판매 중 저자
- **mid 후보**: "추천책" 섹션 + "전자책" 탭에서 추천수 100~1,000 저자
- **underdog 후보**: "공개책" + "최신순" 탭에서 추천수 10~100, 주간 조회수 50+ 저자

### 제외 기준 (스크리닝)

1. 이미 종이책 출판 계약된 저자 (책 상단에 출판 고지 있는지 확인)
2. 미완결 & 최근 1년 이상 업데이트 없음 (좀비 책)
3. 분량이 30페이지 이하 (책이라기엔 너무 얇음)
4. 번역서 (저자 권한 애매)

### 후보 추천 장르 (업무 노하우 포지셔닝과 정렬)

**강한 정렬** (우선)
- 개발 실무 (코드 리뷰, 리팩토링, 테스팅, 배포 등)
- 데이터 사이언스 실무 (현업 사례 기반)
- DevOps / SRE / 인프라 운영 노하우
- 프로젝트 관리 / PM 실무
- 특정 도구 마스터리 (Git, Docker, AWS 등)

**약한 정렬** (선별적)
- 언어 입문서 (너무 기초는 지양, "현업에서 쓰는 OO" 각도면 OK)
- 알고리즘 / CS 이론 (시험·취업 타겟이면 선별 수용)

**정렬 낮음** (제외 권장)
- 순수 종교 자료
- 개인 일기·에세이
- GIS 등 극단적 니치 (TAM 작음)

## 2. 프로파일링 컬럼 가이드 (CSV)

| 컬럼 | 설명 | 예시 / 가능한 값 |
|---|---|---|
| `tier` | 티어 분류 | `anchor` / `mid` / `underdog` |
| `priority` | 영입 우선순위 | `1` (최우선) / `2` (대기) / `3` (관망) |
| `author_name` | 저자명 | 위키독스 표기 그대로 |
| `book_title` | 대표 책 제목 | 위키독스 표기 그대로 |
| `book_id` | 책 ID | URL의 숫자 (예: `19104`) |
| `book_url` | 책 URL | `https://wikidocs.net/book/{id}` |
| `author_profile_url` | 저자 프로필 | `https://wikidocs.net/profile/info/book/{authorId}` |
| `weekly_views` | 주간 조회수 | 책 페이지에 노출 |
| `total_recommends` | 누적 추천수 | 책 페이지에 노출 |
| `has_ebook` | 전자책 판매 중 | `Y` / `N` — `/buy/ebook/{id}` 경로 존재 시 Y |
| `topic_category` | 주제 대분류 | `dev` / `data` / `devops` / `pm` / `design` / `non-tech` |
| `topic_detail` | 주제 상세 | 자유 기술 (예: "FastAPI 실무 배포") |
| `completion_status` | 완결 여부 | `complete` / `in-progress` / `stale` (1년+ 방치) |
| `chapter_count` | 챕터/페이지 수 | 대략 숫자 (분량 감) |
| `last_updated` | 최근 업데이트 | YYYY-MM 형식 (책 내부 확인) |
| `author_external_links` | 외부 링크 | 블로그/GitHub/링크드인 URL (프로필에서 수집) |
| `author_role_estimate` | 실무 배경 추정 | `senior-dev` / `junior-dev` / `instructor` / `researcher` / `unknown` |
| `offer_type` | 영입 오퍼 | `A` (프리미엄 개정판) / `B` (신규 업무 노하우 집필) / `C` (인터랙티브 전환만) |
| `contact_channel` | 컨택 채널 | `wikidocs-msg` / `blog-comment` / `email` / `linkedin` / `twitter` |
| `contact_status` | 컨택 상태 | `not-contacted` / `contacted` / `replied` / `interested` / `agreed` / `declined` |
| `first_contact_date` | 첫 컨택일 | YYYY-MM-DD |
| `notes` | 비고 | 자유 기술 (출판 계약 여부, 관심사, 특이사항) |

## 3. 영입 우선순위 판정 기준

`priority = 1` (지금 즉시 컨택)
- 실무 경력이 명확히 보이는 저자 (블로그/링크드인에서 확인)
- 최근 6개월 내 업데이트
- 외부 연락 채널 확보됨 (블로그 댓글, 이메일 등)
- 장르가 "업무 노하우"와 강하게 정렬

`priority = 2` (준비 후 컨택)
- 퀄리티는 좋은데 저자 이력/외부 채널 파악 필요
- 콜드 메일보다는 블로그/SNS 팔로잉 후 자연스러운 접점 만들기

`priority = 3` (관망)
- 저자 정체 불명 / 활동 중단 상태 / 장르 애매

## 4. 오퍼 타입별 핵심 카피

### 오퍼 A — "프리미엄 개정판"
> "당신이 위키독스에 올린 [책 제목]을 publedge에서 유료 개정판으로 함께 발행하지 않으시겠어요? 기존 위키독스 무료판은 그대로 두시고, publedge판에는 **인터랙티브 워크시트·체크리스트·실습 템플릿**을 덧입혀 독자가 '읽는 책'이 아니라 '적용하는 책'으로 체험할 수 있게 만듭니다. 초기 10명 한정 **수수료 5% 평생 고정** + 인터랙티브 전환은 저희가 도와드립니다."

### 오퍼 B — "업무 노하우 신규 집필"
> "[책 제목]을 완주해내신 이력을 인상 깊게 보았습니다. 이번엔 기술서가 아니라, [저자 실무 영역]에서 쌓으신 **진짜 업무 노하우**를 책으로 엮어보시는 건 어떠세요? 예를 들어 '[저자 특화 주제]' 같은 주제로요. 1:1 에디터가 붙어서 목차 설계부터 출간까지 함께하고, 초기 10명 한정 수수료 5% 평생 고정 조건입니다."

### 오퍼 C — "인터랙티브 전환 대행"
> "당신의 [책 제목]을 publedge에서 전자책으로 유료 발행하는 건 어떠세요? 원고는 그대로 가져오시고, 저희가 **인터랙티브 요소·표지 디자인·판매 페이지·초기 마케팅**을 전부 대행합니다. 저자분의 시간 투입은 검수 단계에만 있으면 됩니다. 초기 입점 10명 수수료 5% 평생 고정."

## 5. 컨택 프로토콜

### 순서

1. **관찰** (1~2주) — 저자 블로그/SNS 팔로우, 댓글 몇 번 남기며 존재 인지시키기
2. **콜드 컨택** — 위키독스 방명록 또는 블로그 댓글로 "publedge 운영자 이기창입니다. 조심스럽게 제안드릴 게 있어서..." 로 시작
3. **응답 시 정식 레터** — 위 오퍼 A/B/C 중 맞춤 선택해서 1페이지 이내 세일즈
4. **미팅 제안** — 15~30분 줌/구글 미팅 또는 오프라인 (수도권 한정)
5. **계약·온보딩** — 표준 계약서 + 1:1 제작 워크숍 킥오프

### 첫 메시지 톤

- ❌ "저희 플랫폼이 좋아서요~" (광고 톤)
- ❌ "수수료 5%!" (혜택 프론트)
- ✅ "[구체 챕터명/문장 인용]에서 이런 부분이 인상 깊었고, [저자 역량]을 보고 제안드립니다"

**구체성 = 성실함 = 응답률**. 아무 저자한테나 뿌린다는 인상을 주면 즉시 무시당해요.

### 응답 없을 때

- 1차 컨택 → 2주 기다림 → 2차 리마인드 1회 → 그 이후 종료
- 공격적 반복 금지 (커뮤니티 평판 리스크)

## 6. 운영 팁

- **CSV 열기**: Google Sheets 또는 Numbers로 import해서 작업 추천 (필터·정렬 편함)
- **업데이트 주기**: 매주 금요일에 `contact_status` 업데이트 + 다음 주 컨택 후보 5명 선정
- **목표 페이스**: 주당 5명 컨택 → 응답률 20% 가정 시 월 4명 응답 → 월 1~2명 계약까지 도달
- **번아웃 방지**: 모든 저자한테 맞춤형으로 쓰면 지침. 상위 10명은 풀 맞춤, 나머지는 세미-템플릿(변수 3~5개만 개인화)
