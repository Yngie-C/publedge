# 오퍼 A — 프리미엄 개정판 영입 레터

## 용도

**대상**: 이미 위키독스에 완결된 책(특히 전자책 판매 중이거나 추천수 100+인 책)을 가진 저자
**제안**: 위키독스 무료판은 유지하고, publedge에서 인터랙티브 요소 추가한 **유료 프리미엄판**을 별도 발행
**핵심 메시지**: "당신이 이미 쓴 책을 한 단계 업그레이드해서 수익화"

---

## 개인화 변수 (발송 전 치환 필수)

| 변수 | 내용 | 예시 |
|---|---|---|
| `{{author_name}}` | 저자명 | "이민재" |
| `{{book_title}}` | 대표 책 제목 | "실무자를 위한 FastAPI" |
| `{{book_url}}` | 위키독스 책 URL | https://wikidocs.net/book/12345 |
| `{{specific_chapter}}` | 인상 깊은 챕터명 | "3장 의존성 주입 실전" |
| `{{specific_insight}}` | 구체적 인용 또는 배운 점 | "레이어드 아키텍처에서 DI 컨테이너 바인딩 순서" |
| `{{author_role_guess}}` | 저자의 실무 배경 추정 | "현업 백엔드 엔지니어로서" |
| `{{your_name}}` | 발신자 이름 | (본인 이름) |
| `{{platform_url}}` | publedge 공개 URL | https://publedge.kr 등 |
| `{{sample_book_url}}` | 쇼케이스 책 링크 | 자소서 파이프라인 or 포트폴리오 바이브코딩 |

## 버전 1 — 짧은 첫 컨택 (위키독스 방명록 / 블로그 댓글 / 첫 DM)

**분량 목표**: 300자 이내

```
안녕하세요, {{author_name}}님.

publedge라는 인터랙티브 전자책 플랫폼을 만들고 있는 {{your_name}}입니다.

{{book_title}}의 {{specific_chapter}}에서 {{specific_insight}} 부분이 특히 인상 깊었습니다. {{author_role_guess}} 쌓으신 실전 감각이 글에 진하게 묻어난 책이라 생각합니다.

조심스럽게 제안드릴 것이 있어 연락드립니다. 저희 플랫폼에서 {{book_title}}의 프리미엄 개정판을 함께 발행하는 구조인데, 위키독스 무료판은 그대로 두시고 publedge판에는 인터랙티브 워크시트·체크리스트·실습 템플릿을 덧입혀 "읽는 책"을 "적용하는 책"으로 바꾸는 개념입니다.

초기 입점 10명 한정 수수료 5% 평생 고정 + 인터랙티브 전환 작업은 저희가 전담합니다. 15분 정도 더 설명드릴 기회를 주시면 감사하겠습니다.

불편하신 제안이라면 답변 안 주셔도 괜찮습니다.

{{your_name}} 드림 · publedge
```

## 버전 2 — 정식 제안 이메일 (1차 응답 있을 때 발송)

**제목 예시** (A/B 테스트용):
- `{{author_name}}님께 — publedge 프리미엄 개정판 제안 드립니다`
- `[publedge] {{book_title}} 인터랙티브판 발행 제안`
- `{{book_title}} 독자층 확장 제안 (publedge {{your_name}})`

**본문 분량 목표**: 450~550자 (스크롤 없이 한 화면)

```
{{author_name}}님, 안녕하세요.

앞서 답신 주셔서 감사합니다. 제안 드린 내용을 좀 더 자세히 풀어드리겠습니다.

■ publedge가 뭔지 간단히

publedge는 "읽는 전자책"을 "적용하는 워크북"으로 바꾸는 데 특화된 인터랙티브 전자책 플랫폼입니다. 체크리스트·워크시트·실습 템플릿·진단 퀴즈 같은 요소를 책 안에 직접 삽입할 수 있고, 독자가 입력한 내용은 개인 저장소에 남아 반복 참조됩니다. 샘플 두 권이 이미 발간 중입니다: {{sample_book_url}}

■ 프리미엄 개정판 제안 구조

1. 위키독스 무료판은 그대로 유지 (저자님 SEO 자산 훼손 없음)
2. publedge에는 "프리미엄 개정판"으로 별도 발행 — 업데이트된 내용 + 인터랙티브 레이어 추가
3. 인터랙티브 전환 작업은 저희가 담당 (저자님 시간 투입은 검수 단계만)
4. 가격은 저자님이 결정 (평균 2~3만원 권장, 5만원 이내)

■ 초기 입점 혜택 (10명 한정)

• 수수료 5% 평생 고정 (일반 정책 대비 대폭 할인)
• 홈 고정 노출 & 공식 소개 포스트
• 론칭 주간 마케팅 전담 (SNS 크로스포스트, 뉴스레터)
• 첫 100권 판매까지 플랫폼이 책임지고 밀어드립니다

■ 다음 스텝 제안

편하신 시간에 줌 15~20분 미팅 가능하실까요? 구체적인 인터랙티브 전환 예시를 보여드리면서 {{book_title}}에 어떻게 적용될지 함께 스케치해보면 좋을 것 같습니다. 다음 주 평일 중 편하신 시간대 두세 개 주시면 맞춰 드립니다.

제안이 맞지 않으시거나 지금이 타이밍이 아니라 판단되시면 가볍게 거절 주셔도 괜찮습니다.

감사합니다.

{{your_name}} 드림
publedge 대표 · lkc0514@gmail.com
{{platform_url}}
```

## 톤 가이드

**✅ 하기**
- 저자의 구체적 챕터·문장 인용 (성실함 = 응답률)
- "조심스럽게", "제안이 맞지 않으시면 거절 주셔도 된다" 류의 여지
- 혜택을 구체 숫자로 (5%, 10명 한정, 100권)
- 다음 스텝을 **낮은 커밋먼트**로 (15분 미팅, 계약서 X)

**❌ 피하기**
- "혁신적인", "독보적인", "최고의" 류 마케팅 클리셰
- "지금 놓치면 안 됩니다" 류의 FOMO 압박
- 이모지 2개 이상 (특히 🎉✨🔥 계열은 스팸 인상)
- 답장 재촉 ("빠른 답변 부탁드립니다")
- 위키독스 커뮤니티·플랫폼 비하 (절대 금물)

## 발송 후 팔로업 프로토콜

| 상황 | 행동 |
|---|---|
| 답 없음 (2주) | 리마인드 1회, "혹시 놓치셨나 싶어..." 가볍게 |
| 리마인드 후 답 없음 | 종료. 6개월 후 재시도 가능 |
| "관심 없다" 정중 거절 | 감사 메시지 + 향후 변경 시 언제든 연락 달라 |
| "계약 중이다/바쁘다" | 감사 + "여유 생기면 알려달라", 3개월 후 리핑 |
| 미팅 수락 | 48시간 내 캘린더 확정 + 어젠다 공유 |

## 실패 사례에서 배우는 함정

- **"개정판 발행 시 기존 독자가 '무료판을 왜 유지하지 않냐'고 화낸다"** → 무료판은 반드시 유지하는 구조로 설계. 레터에 이 점 먼저 안심시키기.
- **"내 책이 왜 당신 플랫폼에 어울리는지 구체적 이유가 없다"** → `{{specific_insight}}` 부분을 절대 생략 말 것. 이게 빠지면 스팸으로 읽힘.
- **"수수료 5%보다 출판사가 인세 10% 주는 게 더 낫지 않나?"** → 종이책 인세와 전자책 수수료 계산 구조가 다름을 설명 가능하도록 준비 (2만원 책 × 수수료 5% = 저자 수취 19,000원 / 2만원 책 × 인세 10% = 저자 수취 2,000원).
